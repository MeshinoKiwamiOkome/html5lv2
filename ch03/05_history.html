① History API

pushState / replaceState とは

History API を使うと、ページを再読み込みせずにURLを変更できます。 これが SPA（シングルページアプリケーション）のルーティングの核心です。 pushState は新しい履歴エントリを追加し、 replaceState は現在のエントリを置き換えます。 

// pushState(state, title, url)
// 新しい履歴エントリを追加してURLを変更（ページ遷移なし）
history.pushState(
  { page: 'home', id: 1 }, // state オブジェクト（任意のデータ）
  '',                          // title（現在のブラウザは無視）
  '/home'                      // 新しいURL（同一オリジンのみ）
);

// replaceState：現在の履歴エントリを置き換える（戻るボタンに残らない）
history.replaceState(
  { page: 'home', updated: true },
  '',
  '/home?tab=profile'
);

// 履歴の移動
history.back();      // 1つ戻る（ブラウザの「戻る」ボタンと同じ）
history.forward();   // 1つ進む
history.go(-2);      // 2つ戻る
history.go(1);       // 1つ進む
history.go(0);       // 現在のページをリロード

// 履歴の長さと現在の state
console.log(history.length); // → 履歴エントリ数
console.log(history.state);  // → 現在の state オブジェクト

// popstate イベント：戻る・進むボタン押下時に発火
window.addEventListener('popstate', (event) => {
  console.log('popstate 発火！');
  console.log(event.state); // → pushState で渡した state
  // ここでページの内容を切り替える（SPAのルーティング）
  renderPage(event.state);
});
📌 試験頻出：pushState vs replaceState の違い pushState：新しいエントリをスタックに追加する。戻るボタンで前のページに戻れる。
replaceState：現在のエントリを置き換える。戻るボタンに残らない。
どちらも popstate イベントは発火しません（popstate は back/forward/go でのみ発火）。
メソッド履歴スタックpopstate 発火主な用途
pushState新エントリを追加❌ 発火しないページ遷移のシミュレート
replaceState現在のエントリを置換❌ 発火しないURLの更新（履歴に残さない）
back() / forward() / go()ポインタを移動✅ 発火するプログラムによる履歴移動
⚠️ pushState の URL 制限 pushState で指定できる URL は同一オリジンのみです。 異なるオリジンの URL を指定すると SecurityError が発生します。 また、相対URLも使用可能です（例：'../about'）。

② Location API

URL の構成要素と Location プロパティ

window.location オブジェクトは現在のURLの情報を提供し、 ナビゲーションの制御もできます。URLは以下の構成要素に分解されます。 

// URL: https://example.com:8080/path/page?q=hello&lang=ja#section1

console.log(location.href);      // → 'https://example.com:8080/path/page?q=hello&lang=ja#section1'
console.log(location.protocol);  // → 'https:'（コロン含む）
console.log(location.host);      // → 'example.com:8080'（ポート含む）
console.log(location.hostname);  // → 'example.com'（ポートなし）
console.log(location.port);      // → '8080'
console.log(location.pathname);  // → '/path/page'
console.log(location.search);    // → '?q=hello&lang=ja'（?含む）
console.log(location.hash);      // → '#section1'（#含む）
console.log(location.origin);    // → 'https://example.com:8080'

// ナビゲーション
location.assign('https://example.com');  // 遷移（履歴に残る）
location.replace('https://example.com'); // 遷移（履歴に残らない）
location.reload();                       // 現在のページをリロード
location.href = 'https://example.com';    // assign と同じ

// URLSearchParams でクエリ文字列を解析
const params = new URLSearchParams(location.search);
console.log(params.get('q'));    // → 'hello'
console.log(params.get('lang')); // → 'ja'
params.set('q', 'world');
console.log(params.toString()); // → 'q=world&lang=ja'
プロパティ/メソッド説明履歴への影響
location.href = urlURLに遷移履歴に追加
location.assign(url)URLに遷移履歴に追加
location.replace(url)URLに遷移（置換）履歴に残らない
location.reload()現在のページをリロード変化なし
location.hash = '#id'ハッシュを変更履歴に追加
📌 試験頻出：assign vs replace の違い location.assign(url)：遷移後に戻るボタンで前のページに戻れる（履歴に追加）
location.replace(url)：遷移後に戻るボタンで戻れない（現在のエントリを置換）
ログインページからダッシュボードへの遷移など、「戻れないようにしたい」場合に replace を使います。
