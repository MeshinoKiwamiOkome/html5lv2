① window オブジェクトとグローバルスコープ

window はすべての頂点

window オブジェクトはブラウザ環境におけるグローバルオブジェクトです。 JavaScriptのグローバルスコープで宣言した変数・関数はすべて window のプロパティになります。 また document・location・history・navigator・screen などの重要なオブジェクトもすべて window のプロパティです。 

// window はグローバルオブジェクト
// グローバル変数は window のプロパティになる
var myVar = 'hello';
console.log(window.myVar); // → 'hello'

// window. は省略可能
window.alert('Hi');  // alert('Hi') と同じ
window.console.log('test'); // console.log と同じ

// 主要プロパティ
console.log(window.document);   // DOM のルート
console.log(window.location);   // 現在のURL情報
console.log(window.history);    // ブラウザ履歴
console.log(window.navigator);  // ブラウザ情報
console.log(window.screen);     // 画面情報
console.log(window.innerWidth);  // ビューポート幅
console.log(window.innerHeight); // ビューポート高さ

// let/const はグローバルスコープでも window プロパティにならない
let x = 10;
console.log(window.x); // → undefined（var と異なる点）
📌 試験頻出：var vs let/const のグローバル挙動 var で宣言したグローバル変数は window のプロパティになりますが、 let / const で宣言した変数は window のプロパティにはなりません。 これはモジュール設計の観点でも重要な違いです。
プロパティ説明主な用途
window.location現在のURL情報・ナビゲーションURLの取得・ページ遷移
window.historyブラウザの履歴スタックSPA のルーティング
window.navigatorブラウザ・OS情報UA判定・Geolocation
window.screenディスプレイの物理情報解像度・色深度の取得

② タイマー処理

setTimeout / setInterval / clearTimeout / clearInterval

setTimeout は指定時間後に1回だけ処理を実行します。 setInterval は指定間隔で繰り返し処理を実行します。 どちらもタイマーIDを返し、それを使って clearTimeout / clearInterval でキャンセルできます。 

// setTimeout：指定時間後に1回だけ実行
const timerId = setTimeout(() => {
  console.log('2秒後に実行！');
}, 2000); // ミリ秒単位（2000ms = 2秒）

// タイマーをキャンセル
clearTimeout(timerId);

// setInterval：一定間隔で繰り返し実行
let count = 0;
const intervalId = setInterval(() => {
  count++;
  console.log(`${count}回目`);
  if (count >= 5) {
    clearInterval(intervalId); // 5回で停止
  }
}, 1000); // 1秒ごと

// ネストした setTimeout でインターバルを再現（より正確）
function tick() {
  console.log('tick');
  setTimeout(tick, 1000); // 処理完了後に次をスケジュール
}
⚠️ setInterval の注意点：遅延の蓄積 setInterval はコールバックの実行時間に関わらず一定間隔でキューに積むため、 処理が重い場合に遅延が蓄積することがあります。 精度が必要な場合はネストした setTimeout を使うと、 前の処理完了後に次をスケジュールするため遅延が蓄積しません。

requestAnimationFrame との違い

requestAnimationFrame(callback)（略称：rAF）は ブラウザの描画タイミングに同期してコールバックを実行します。 通常は約60fps（16.7ms間隔）で呼ばれ、アニメーションに最適です。 タブが非アクティブになると自動的に停止するため、バッテリー効率も優れています。 

// requestAnimationFrame を使ったアニメーション
let rafId;
let x = 0;

function animate(timestamp) {
  // timestamp：ページ読み込みからの経過時間(ms)
  x = (x + 2) % 300;
  ball.style.left = x + 'px';

  rafId = requestAnimationFrame(animate); // 次フレームを予約
}

rafId = requestAnimationFrame(animate); // 開始
cancelAnimationFrame(rafId);             // 停止

// setInterval との比較
// setInterval(animate, 16)  ← 固定間隔（ブラウザ描画と非同期）
// requestAnimationFrame     ← 描画タイミングに同期（滑らか・省電力）
比較項目setInterval(fn, 16)requestAnimationFrame
実行タイミング固定間隔（ブラウザ描画と非同期）ブラウザ描画直前（同期）
フレームレート指定間隔に依存（ズレあり）ディスプレイのリフレッシュレートに追従
非アクティブタブバックグラウンドでも実行自動停止（省電力）
アニメーション品質カクつきやすい滑らか
キャンセルclearInterval(id)cancelAnimationFrame(id)
📌 試験頻出：rAF のコールバック引数 requestAnimationFrame のコールバックには DOMHighResTimeStamp（ページ読み込みからの経過ミリ秒）が渡されます。 これを使って経過時間ベースのアニメーション（時間依存アニメーション）を実装できます。

③ postMessage によるクロスオリジン通信

window.postMessage(data, targetOrigin) は 異なるオリジン間でも安全にメッセージを送受信できる仕組みです。 iframe・別ウィンドウ・Web Worker との通信に使われます。 受信側は message イベントをリッスンします。 

// 送信側（親ウィンドウ）
const iframe = document.querySelector('iframe');

iframe.contentWindow.postMessage(
  { type: 'greeting', text: 'こんにちは' }, // 送信データ
  'https://child.example.com'               // 送信先オリジン（'*' は全許可）
);

// 受信側（iframe 内）
window.addEventListener('message', (event) => {
  // ① オリジンを必ず検証する（セキュリティ上重要！）
  if (event.origin !== 'https://parent.example.com') return;

  console.log(event.data);    // → { type: 'greeting', text: 'こんにちは' }
  console.log(event.origin);  // → 'https://parent.example.com'
  console.log(event.source);  // → 送信元の window オブジェクト

  // ② 返信（event.source を使って送信元に返す）
  event.source.postMessage(
    { type: 'reply', text: '受信しました' },
    event.origin
  );
});
⚠️ postMessage のセキュリティ注意事項 ① 受信側では必ず event.origin を検証してください（信頼できるオリジンのみ処理）
② 送信側の targetOrigin に '*' を指定すると任意のオリジンに送信されるため、 機密データを含む場合は必ず具体的なオリジンを指定してください

④ ダイアログ（alert / confirm / prompt）

メソッド説明返り値
alert(msg)メッセージを表示（OKのみ）undefined
confirm(msg)OK / キャンセルの確認ダイアログtrue / false
prompt(msg, default)テキスト入力ダイアログ入力文字列 / null
 
// alert：通知
alert('処理が完了しました'); // 返り値なし

// confirm：確認
const ok = confirm('削除しますか？');
if (ok) {
  console.log('削除しました');  // OK → true
} else {
  console.log('キャンセル');   // キャンセル → false
}

// prompt：入力
const name = prompt('名前を入力してください', 'デフォルト値');
if (name !== null) {
  console.log(`こんにちは、${name}さん`);
} // キャンセル時は null が返る
💡 モダン開発でのダイアログ alert/confirm/prompt はUIスレッドをブロックするため、 現代のWebアプリではカスタムモーダルダイアログを使うことが推奨されます。 ただし試験では動作・返り値の理解が問われます。
