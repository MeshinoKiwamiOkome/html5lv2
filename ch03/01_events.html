① イベントドリブンモデル

イベントドリブンとは何か

JavaScriptはブラウザ上でイベントドリブン（Event-Driven）モデルで動作します。 これは「何かが起きたとき（イベント発生時）に、あらかじめ登録した処理（ハンドラ）を実行する」という仕組みです。 プログラムが上から下へ順番に実行されるのではなく、ユーザーの操作や状態変化を待ち受けて反応するという点が特徴です。

イベントの種類主なイベント名発生タイミング
マウスclick / dblclick / mouseover / mouseout / mousemoveマウス操作時
キーボードkeydown / keyup / keypressキー操作時
フォームsubmit / change / input / focus / blurフォーム操作時
ドキュメントDOMContentLoaded / loadDOM・リソース読み込み完了時
ウィンドウresize / scroll / beforeunloadウィンドウ操作時
📌 試験頻出ポイント：addEventListener の第3引数 addEventListener(type, listener, useCapture) の第3引数は true でキャプチャフェーズ、false（デフォルト）でバブリングフェーズでリスナーを登録します。 省略した場合は false（バブリング）になります。 
// イベントリスナの登録（基本形）
const btn = document.getElementById('myBtn');

// ① addEventListener（推奨）
btn.addEventListener('click', function(event) {
  console.log('クリックされました', event);
});

// ② アロー関数でも書ける
btn.addEventListener('click', (e) => {
  console.log('イベントタイプ:', e.type); // → 'click'
});

// ③ イベントリスナの削除（同じ関数参照が必要）
function handleClick(e) {
  console.log('クリック');
}
btn.addEventListener('click', handleClick);
btn.removeEventListener('click', handleClick); // 削除

// ④ once オプション：1回だけ実行して自動削除
btn.addEventListener('click', handler, { once: true });
⚠️ removeEventListener の注意点 アロー関数や無名関数を直接渡すと削除できません。削除したい場合は必ず名前付き関数または変数に格納した関数参照を使ってください。
btn.removeEventListener('click', () => {}) は削除されません（別の関数オブジェクトとして扱われるため）。

イベントオブジェクトの主要プロパティ

プロパティ説明例
event.typeイベントの種類'click'
event.targetイベントが発生した要素（クリックされた要素）クリックされた <button>
event.currentTargetリスナーが登録されている要素リスナーを持つ親要素
event.bubblesバブリングするかどうかtrue / false
event.cancelableデフォルト動作をキャンセルできるかtrue / false
event.timeStampイベント発生時刻（ms）1234.56
event.key押されたキー（KeyboardEvent）'Enter'
event.clientX/Yビューポート基準のマウス座標120, 340
💡 target vs currentTarget の違い event.target は「実際にクリックされた要素」、 event.currentTarget は「リスナーが登録された要素」です。 イベント委譲では親要素にリスナーを登録するため、この2つが異なる要素になります。

② イベントの伝播（バブリング・キャプチャリング）

DOMイベントは3つのフェーズで伝播します。 ①キャプチャフェーズ（window → 対象要素へ下降）、 ②ターゲットフェーズ（対象要素で発火）、 ③バブリングフェーズ（対象要素 → window へ上昇）です。 デフォルトではバブリングフェーズでリスナーが呼ばれます。 

// バブリング：子 → 親 → 祖先 の順に伝播（デフォルト）
document.querySelector('.outer').addEventListener('click', (e) => {
  console.log('outer（バブリング）');
  // e.stopPropagation() で伝播を止められる
});

// キャプチャリング：祖先 → 親 → 子 の順（第3引数 true）
document.querySelector('.outer').addEventListener('click', (e) => {
  console.log('outer（キャプチャ）');
}, true); // ← true でキャプチャフェーズ

// stopPropagation：それ以上の伝播を止める
inner.addEventListener('click', (e) => {
  e.stopPropagation(); // 親へのバブリングを止める
});

// stopImmediatePropagation：同一要素の残りのリスナーも止める
inner.addEventListener('click', (e) => {
  e.stopImmediatePropagation();
});
📌 試験頻出：伝播の順序 クリック時の実行順序：
キャプチャ（外→内）→ ターゲット → バブリング（内→外）
同一要素に両方登録した場合、キャプチャリスナーが先に実行されます。

③ preventDefault() の使い方

event.preventDefault() はブラウザのデフォルト動作をキャンセルします。 例えば、リンクのページ遷移・フォームの送信・チェックボックスのチェック切り替えなどを JavaScriptで制御したい場合に使います。 伝播は止めません（伝播を止めるのは stopPropagation()）。 

// リンクのデフォルト遷移をキャンセル
link.addEventListener('click', (e) => {
  e.preventDefault(); // ページ遷移しない
  console.log('リンクがクリックされましたが遷移しません');
});

// フォーム送信をキャンセルしてバリデーション
form.addEventListener('submit', (e) => {
  e.preventDefault(); // ページリロードを防ぐ
  // 独自のバリデーション処理
  if (isValid()) form.submit();
});

// 数字のみ入力を許可（それ以外のキーをキャンセル）
input.addEventListener('keydown', (e) => {
  if (!/[0-9]/.test(e.key) && e.key !== 'Backspace') {
    e.preventDefault();
  }
});
⚠️ passive イベントリスナと preventDefault addEventListener('touchstart', handler, { passive: true }) のように passive: true を指定すると、そのリスナー内で preventDefault() を 呼んでも無効になります（スクロールパフォーマンス最適化のため）。

④ イベント委譲（Event Delegation）

イベント委譲とは、子要素ひとつひとつにリスナーを登録するのではなく、 親要素にまとめてリスナーを登録するテクニックです。 バブリングを利用して、動的に追加された要素にも対応できます。 

// ❌ 非効率：各 li にリスナーを登録
document.querySelectorAll('li').forEach(li => {
  li.addEventListener('click', handler); // 要素数分のリスナーが生成される
});

// ✅ 効率的：親 ul にリスナーを1つだけ登録（イベント委譲）
const ul = document.querySelector('ul');
ul.addEventListener('click', (e) => {
  // クリックされた要素が li かどうかを確認
  if (e.target.matches('li')) {
    console.log('クリックされた li:', e.target.textContent);
  }
});

// 動的に追加した要素にも自動で反応する！
const newLi = document.createElement('li');
newLi.textContent = '新しいアイテム';
ul.appendChild(newLi); // 追加後もクリックで反応する
📌 イベント委譲のメリット ① メモリ効率が良い（リスナー数が少ない）
② 動的に追加した要素にも自動対応
③ コードがシンプルになる
注意：stopPropagation() が子要素で呼ばれると委譲が機能しなくなります。

⑤ カスタムイベントの作成と発火

CustomEvent コンストラクタを使うと、独自のイベントを作成できます。 dispatchEvent() で発火し、通常の addEventListener() で受信できます。 コンポーネント間の疎結合な通信に役立ちます。 

// ① カスタムイベントの作成
const myEvent = new CustomEvent('myCustomEvent', {
  detail: { message: 'こんにちは', value: 42 }, // 任意のデータを渡せる
  bubbles: true,    // バブリングするか
  cancelable: true  // preventDefault() できるか
});

// ② リスナーを登録（受信側）
document.addEventListener('myCustomEvent', (e) => {
  console.log('受信！', e.detail.message); // → '受信！こんにちは'
  console.log('値:', e.detail.value);       // → '値: 42'
});

// ③ イベントを発火（送信側）
document.dispatchEvent(myEvent);

// 特定の要素から発火することもできる
const btn = document.getElementById('myBtn');
btn.dispatchEvent(new CustomEvent('activate', {
  detail: { source: 'button' },
  bubbles: true
}));
💡 CustomEvent の活用シーン ・コンポーネント間の通信（親子関係のない要素同士）
・ライブラリやプラグインのフック機能
・状態変化の通知（ショッピングカートに商品が追加されたなど）
