① CSSセレクタの種類

querySelector で使えるセレクタ一覧

querySelector / querySelectorAll には CSSで使えるセレクタをそのまま指定できます。 複雑なセレクタも使えるため、従来の getElementById などより柔軟です。

セレクタの種類記法説明例
要素セレクタtagタグ名で選択'p' 'div'
クラスセレクタ.classクラス名で選択'.btn'
IDセレクタ#idIDで選択'#header'
属性セレクタ[attr]属性の有無・値で選択'[type="text"]'
擬似クラス:pseudo状態・位置で選択':first-child' ':hover'
子孫結合子A BAの子孫のB'nav a'
子結合子A > BAの直接の子のB'ul > li'
隣接兄弟A + BAの直後の兄弟B'h2 + p'
一般兄弟A ~ BAの後の兄弟B全て'h2 ~ p'
複合セレクタA, BAまたはB'h1, h2, h3'
 
// 様々なセレクタの使用例
document.querySelector('p');                   // 最初の p 要素
document.querySelector('.btn-primary');        // クラス
document.querySelector('#main-title');         // ID
document.querySelector('input[type="email"]'); // 属性
document.querySelector('li:first-child');      // 擬似クラス
document.querySelector('nav > ul > li');       // 子結合子
document.querySelector('.card:not(.disabled)'); // :not() 擬似クラス

// querySelectorAll で全要素取得
document.querySelectorAll('ul.menu li');       // メニューの全 li
document.querySelectorAll('[data-active="true"]'); // data属性
document.querySelectorAll('h1, h2, h3');        // 複数セレクタ

// スコープを絞って検索（要素.querySelector）
const form = document.getElementById('myForm');
const inputs = form.querySelectorAll('input'); // form 内の input のみ

② 取得メソッドの比較

querySelector vs 従来のメソッド

メソッド引数返り値静的/動的速度
getElementById ID文字列Element / null— ⚡ 最速
getElementsByClassName クラス名HTMLCollection🔄 動的 🚀 高速
getElementsByTagName タグ名HTMLCollection🔄 動的 🚀 高速
querySelector CSSセレクタElement / null— 🐢 やや遅い
querySelectorAll CSSセレクタNodeList📸 静的 🐢 やや遅い
📌 試験頻出：静的 NodeList vs 動的 HTMLCollection querySelectorAll が返す NodeList は静的スナップショットです。 取得後にDOMに要素を追加・削除しても NodeList の内容は変わりません。
getElementsByClassName などが返す HTMLCollection は動的です。 DOMが変わるとリアルタイムに内容が更新されます。 これは無限ループの原因になることもあるため注意が必要です。 
// NodeList（静的）vs HTMLCollection（動的）の違い
const ul = document.querySelector('ul');

// 静的 NodeList：取得時点のスナップショット
const nodeList = document.querySelectorAll('li');
console.log(nodeList.length); // → 3

// 動的 HTMLCollection：DOMの変化を反映
const htmlCol = document.getElementsByTagName('li');
console.log(htmlCol.length); // → 3

// li を1つ追加
const newLi = document.createElement('li');
ul.appendChild(newLi);

console.log(nodeList.length); // → 3（変わらない！静的）
console.log(htmlCol.length);  // → 4（自動更新！動的）

// ⚠️ HTMLCollection の無限ループ罠
// for ループ中に要素を追加すると length が増え続ける！
const items = document.getElementsByClassName('item');
// for (let i = 0; i < items.length; i++) {  ← 危険！
//   items[i].classList.add('item'); // item が増え続ける
// }
// → Array.from() で静的コピーを作ってから操作する
Array.from(items).forEach(el => el.classList.add('processed'));
⚠️ HTMLCollection の無限ループ罠 getElementsByClassName などで取得した HTMLCollection を for ループで処理中に同じクラスを追加すると、 length が増え続けて無限ループになります。 安全に処理するには Array.from(collection) や [...collection] で静的配列に変換してから操作しましょう。

③ closest() と matches() の活用

closest()：祖先要素を遡って検索

element.closest(selector) は、要素自身から始めて DOM ツリーを上方向（祖先方向）に遡り、 セレクタに一致する最初の要素を返します。 一致するものがなければ null を返します。 イベント委譲と組み合わせて使うことが多いです。 

// closest()：自分から祖先方向に遡って検索
const btn = document.querySelector('.delete-btn');

// ボタンの祖先の .card 要素を取得
const card = btn.closest('.card');
if (card) {
  card.remove(); // カードごと削除
}

// イベント委譲との組み合わせ（実践的な使い方）
document.addEventListener('click', (e) => {
  // クリックされた要素の祖先に .card があるか確認
  const card = e.target.closest('.card');
  if (card) {
    console.log('カードがクリックされました', card);
  }
});

// matches()：要素がセレクタに一致するか確認
const el = document.querySelector('#myBtn');
console.log(el.matches('.btn'));          // → true/false
console.log(el.matches(':disabled'));     // → true/false
console.log(el.matches('button.primary')); // → true/false

// イベント委譲での matches() 活用
list.addEventListener('click', (e) => {
  if (e.target.matches('li.item')) {
    console.log('リストアイテムがクリックされた');
  }
});
メソッド検索方向返り値主な用途
closest(sel) 自身 → 上方向（祖先） Element / null 親コンテナの取得、イベント委譲
matches(sel) 自身のみ確認 true / false 条件分岐、イベント委譲のフィルタリング
querySelector(sel) 下方向（子孫） Element / null 子孫要素の取得
📌 試験頻出：closest() の検索開始位置 closest() は要素自身も含めて上方向に検索します。 つまり要素自身がセレクタに一致する場合は、その要素自身が返されます。 querySelector() は下方向（子孫）への検索で、自身は含みません。
