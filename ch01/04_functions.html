① 関数の基本

重要ポイント：関数は処理をひとまとめにして名前をつけたものです。function キーワードで定義し、関数名() で呼び出します。
同じ処理を何度も書かずに済み、コードの再利用性が上がります。

関数の構造

🔧 定義

📥 引数
name

⚙️ 処理

📤 戻り値

// 関数の定義（function 宣言）
function greet() {
  console.log("こんにちは！");
}

// 関数の呼び出し
greet(); // → こんにちは！
greet(); // → こんにちは！（何度でも呼び出せる）

// 関数は定義より前に呼び出せる（ホイスティング）
sayHello(); // → Hello!（定義前でも動く）
function sayHello() {
  console.log("Hello!");
}

② 引数の基本・複数の引数

重要ポイント：引数（パラメータ）は関数に渡す入力値です。複数の引数はカンマで区切って定義します。
引数を渡さなかった場合は undefined になります。デフォルト値を設定することもできます。

← 値と演算子を選んで実行してみましょう
// 引数を1つ受け取る関数
function greet(name) {
  console.log(`こんにちは、${name}さん！`);
}
greet("Alice"); // → こんにちは、Aliceさん！
greet("Bob");   // → こんにちは、Bobさん！
greet();       // → こんにちは、undefinedさん！（引数なし）

// 複数の引数
function add(a, b) {
  return a + b;
}
console.log(add(3, 5));  // → 8
console.log(add(10, 20)); // → 30

// デフォルト引数（ES6〜）
function greetDefault(name = "ゲスト") {
  console.log(`こんにちは、${name}さん！`);
}
greetDefault();         // → こんにちは、ゲストさん！
greetDefault("Alice"); // → こんにちは、Aliceさん！

// 残余引数（...args）：可変長引数
function sum(...nums) {
  return nums.reduce((acc, n) => acc + n, 0);
}
console.log(sum(1, 2, 3));       // → 6
console.log(sum(1, 2, 3, 4, 5)); // → 15

③ 戻り値

重要ポイント：return 文で関数の結果を返します。return に達した時点で関数の実行が終了します。
return がない関数、または値なしの return は undefined を返します。 
// 戻り値のある関数
function square(n) {
  return n * n;  // n の2乗を返す
}
const result = square(5);
console.log(result); // → 25

// return に達したら関数終了（早期リターン）
function divide(a, b) {
  if (b === 0) {
    return "エラー：0で割れません"; // ここで終了
  }
  return a / b;
}
console.log(divide(10, 2)); // → 5
console.log(divide(10, 0)); // → "エラー：0で割れません"

// 複数の値を返す（オブジェクトや配列で）
function minMax(arr) {
  return {
    min: Math.min(...arr),
    max: Math.max(...arr)
  };
}
const { min, max } = minMax([3, 1, 4, 1, 5, 9]);
console.log(min, max); // → 1 9

// return なし → undefined を返す
function noReturn() {
  console.log("処理のみ");
}
console.log(noReturn()); // → "処理のみ" が出力され、undefined が返る

④ さまざまな関数の作り方とその違い

重要ポイント：JavaScriptには関数宣言・関数式・アロー関数など複数の書き方があります。
試験では特にアロー関数と this の扱いの違いが重要です。

関数の種類（クリックでコード例を表示）

最も基本
関数宣言
function fn() { ... }
変数に代入
関数式
const fn = function() { ... };
ES6〜 推奨
アロー関数
const fn = () => { ... };
省略形
アロー関数（短縮）
const fn = x => x * 2; // 1行なら{}とreturn省略
即時実行
IIFE
(function() { ... })();
関数を扱う
高階関数
[1,2,3].map(x => x*2) // 関数を引数に渡す

関数宣言 vs 関数式 vs アロー関数 比較表

特徴関数宣言関数式アロー関数
ホイスティング ✅ あり（定義前に呼び出せる） ❌ なし（定義後のみ） ❌ なし（定義後のみ）
this の扱い 呼び出し方で変わる 呼び出し方で変わる 外側のスコープの this を継承
arguments オブジェクト ✅ あり ✅ あり ❌ なし（...args を使う）
コンストラクタとして使用 ✅ 可能 ✅ 可能 ❌ 不可（new できない）
推奨場面 トップレベルの関数定義 条件付き定義・コールバック コールバック・メソッド以外

アロー関数の省略ルール

// 通常の関数宣言
function add(a, b) {
  return a + b;
}

// アロー関数（基本形）
const add = (a, b) => {
  return a + b;
};

console.log(add(3, 5)); // → 8

// 引数が1つ → () を省略できる
const double = x => { return x * 2; };

// 引数が0個 → () は省略できない
const greet = () => { console.log("Hello!"); };

// 引数が2個以上 → () は省略できない
const add = (a, b) => { return a + b; };

console.log(double(5)); // → 10

// 本体が1行の式 → {} と return を省略できる
const double = x => x * 2;
const add    = (a, b) => a + b;
const square = n => n ** 2;

console.log(double(5));    // → 10
console.log(add(3, 4));   // → 7
console.log(square(4));   // → 16

// オブジェクトを返す場合は () で囲む
const makeUser = (name, age) => ({ name, age });
console.log(makeUser("Alice", 25)); // → {name:"Alice", age:25}

// 配列メソッドとの組み合わせ（最もよく使う場面）
const nums = [1, 2, 3, 4, 5];
const doubled  = nums.map(n => n * 2);
const evens    = nums.filter(n => n % 2 === 0);
const total    = nums.reduce((acc, n) => acc + n, 0);
console.log(doubled); // → [2,4,6,8,10]
console.log(evens);   // → [2,4]
console.log(total);   // → 15

// 通常の関数：this は呼び出し方で変わる
const obj1 = {
  name: "Alice",
  greet: function() {
    console.log(this.name); // → "Alice"（obj1 を指す）
  }
};
obj1.greet();

// アロー関数：this は外側のスコープを継承する
const obj2 = {
  name: "Bob",
  greet: () => {
    console.log(this.name); // → undefined（外側の this = window）
  }
};
obj2.greet();

// アロー関数が有効な場面：コールバック内での this
function Timer() {
  this.count = 0;
  setInterval(() => {
    this.count++;  // アロー関数なので外側の this（Timer）を参照
    console.log(this.count);
  }, 1000);
}
⚠️ オブジェクトのメソッドにアロー関数を使うと this が期待通りに動きません。
オブジェクトのメソッドには通常の関数を使い、コールバックにはアロー関数を使うのが基本です。

📌 まとめ

項目構文ポイント
関数宣言 function fn(a, b) { return a+b; } ホイスティングあり・定義前に呼び出せる
関数式 const fn = function(a, b) { return a+b; }; 変数に代入・ホイスティングなし
アロー関数（基本） const fn = (a, b) => { return a+b; }; this を継承・コンストラクタ不可
