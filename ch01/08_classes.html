① オブジェクトとメソッド

重要ポイント：オブジェクトにはデータ（プロパティ）だけでなく、 処理（メソッド）も持たせることができます。 メソッド内の this はそのオブジェクト自身を指します。

🎮 オブジェクトビルダー

名前・年齢・職業を入力してオブジェクトを作成しよう
← 値を入力してオブジェクトを作成してみましょう
// オブジェクトリテラルとメソッド
const user = {
  name: "Alice",
  age:  25,
  job:  "エンジニア",

  // メソッド（ES6 短縮記法）
  greet() {
    return `こんにちは！私は${this.name}、${this.age}歳の${this.job}です。`;
  },

  birthday() {
    this.age++;
    return `${this.name}は${this.age}歳になりました！`;
  }
};

console.log(user.greet());    // → "こんにちは！私はAlice、25歳のエンジニアです。"
console.log(user.birthday()); // → "Aliceは26歳になりました！"
console.log(user.age);         // → 26（this.age が変更された）

// this の注意点：アロー関数ではメソッドに使わない
const obj = {
  name: "Bob",
  greet: () => {
    console.log(this.name); // → undefined（this が window を指す）
  }
};

② クラスの基本

重要ポイント：クラスはオブジェクトの「設計図」です。 new キーワードでクラスからインスタンス（実体）を作成します。 constructor は new したときに自動的に呼ばれる初期化メソッドです。

クラス図

📦 class Animal
プロパティ
+ name : string
+ sound : string
メソッド
+ constructor (name, sound)
+ speak (): string
+ toString (): string
↓ new Animal("Dog", "ワン") でインスタンス作成
🐕 alice（Animal インスタンス）
name:"Dog"
sound:"ワン"
🐈 kitty（Animal インスタンス）
name:"Cat"
sound:"ニャン"

🎮 クラスインスタンス作成デモ

Animal クラスのインスタンスを作成してメソッドを呼び出してみよう
← 値を入力してインスタンスを作成してみましょう
// クラスの定義
class Animal {
  // constructor：new したときに自動実行
  constructor(name, sound) {
    this.name  = name;
    this.sound = sound;
  }

  // メソッド
  speak() {
    return `${this.name}が「${this.sound}」と鳴きました`;
  }

  toString() {
    return `Animal { name: "${this.name}", sound: "${this.sound}" }`;
  }
}

// インスタンスの作成
const dog = new Animal("Dog", "ワン");
const cat = new Animal("Cat", "ニャン");

console.log(dog.speak()); // → "Dogが「ワン」と鳴きました"
console.log(cat.speak()); // → "Catが「ニャン」と鳴きました"

// instanceof でクラスの確認
console.log(dog instanceof Animal); // → true

③ 継承（extends / super）

重要ポイント：extends で親クラスを継承し、 super() で親クラスの constructor を呼び出します。 子クラスは親クラスのメソッドをオーバーライド（上書き）できます。

継承の構造

🐾 Animal（親クラス）
name, sound, speak()
↑ extends（継承）
子クラス
🐕 Dog
breed, fetch()
子クラス
🐈 Cat
indoor, purr()

extends / super の使い方

// 親クラス
class Animal {
  constructor(name, sound) {
    this.name  = name;
    this.sound = sound;
  }
  speak() {
    return `${this.name}：「${this.sound}」`;
  }
}

// 子クラス（Animal を継承）
class Dog extends Animal {
  constructor(name, breed) {
    super(name, "ワン"); // 親の constructor を呼ぶ（必須）
    this.breed = breed;  // 子クラス独自のプロパティ
  }

  fetch() {
    return `${this.name}（${this.breed}）がボールを取ってきた！`;
  }
}

const dog = new Dog("ポチ", "柴犬");
console.log(dog.speak()); // → "ポチ：「ワン」"（親のメソッドを使用）
console.log(dog.fetch()); // → "ポチ（柴犬）がボールを取ってきた！"
console.log(dog instanceof Dog);    // → true
console.log(dog instanceof Animal); // → true（親クラスも true）
⚠️ 子クラスの constructor では必ず super() を先に呼ぶこと！
super() を呼ぶ前に this を使うと ReferenceError になります。

メソッドのオーバーライド

class Animal {
  constructor(name) { this.name = name; }
  speak() { return `${this.name}が何か言った`; }
  info()  { return `Animal: ${this.name}`; }
}

class Dog extends Animal {
  // speak をオーバーライド（上書き）
  speak() {
    return `${this.name}：「ワン！」`;
  }

  // super で親のメソッドを呼び出す
  info() {
    return super.info() + " (Dog)"; // 親の info() を拡張
  }
}

class Cat extends Animal {
  speak() {
    return `${this.name}：「ニャン～」`;
  }
}

const animals = [
  new Animal("謎の生き物"),
  new Dog("ポチ"),
  new Cat("タマ"),
];

// ポリモーフィズム：同じメソッド名でも動作が異なる
animals.forEach(a => console.log(a.speak()));
// → "謎の生き物が何か言った"
// → "ポチ：「ワン！」"
// → "タマ：「ニャン～」"

static（静的メソッド・プロパティ）

static とは：インスタンスではなく、クラス自体に属するメソッドやプロパティです。 new せずに クラス名.メソッド() で呼び出せます。 
class MathUtils {
  // static プロパティ
  static PI = 3.14159;

  // static メソッド（インスタンスを作らずに使える）
  static add(a, b) { return a + b; }
  static circleArea(r) { return MathUtils.PI * r ** 2; }
}

// new せずに直接呼び出す
console.log(MathUtils.add(3, 5));        // → 8
console.log(MathUtils.circleArea(5));   // → 78.53975
console.log(MathUtils.PI);             // → 3.14159

// インスタンスカウンター（static の活用例）
class Counter {
  static count = 0;

  constructor() {
    Counter.count++;
    this.id = Counter.count;
  }

  static getCount() { return Counter.count; }
}

new Counter(); new Counter(); new Counter();
console.log(Counter.getCount()); // → 3

getter / setter

getter/setter とは：プロパティのように見えるが、 アクセス時に処理を実行できる特殊なメソッドです。 バリデーションや計算値の提供に使います。 
class Temperature {
  constructor(celsius) {
    this._celsius = celsius; // _ は慣習的にプライベートを示す
  }

  // getter：プロパティのように読み取れる
  get celsius()    { return this._celsius; }
  get fahrenheit() { return this._celsius * 9 / 5 + 32; }
  get kelvin()     { return this._celsius + 273.15; }

  // setter：バリデーション付きで値を設定
  set celsius(val) {
    if (val < -273.15) throw new Error("絶対零度以下は設定できません");
    this._celsius = val;
  }
}

const temp = new Temperature(100);
console.log(temp.celsius);    // → 100（getter）
console.log(temp.fahrenheit); // → 212（getter で計算）
console.log(temp.kelvin);     // → 373.15（getter で計算）

temp.celsius = 0;              // setter を呼び出す
console.log(temp.fahrenheit); // → 32

④ プライベートフィールド（ES2022〜）

重要ポイント：# プレフィックスを付けるとクラス外からアクセスできない プライベートフィールドになります。データの隠蔽（カプセル化）に使います。 
// # プレフィックスでプライベートフィールド
class BankAccount {
  #balance;     // プライベートフィールドの宣言
  #owner;

  constructor(owner, initialBalance) {
    this.#owner   = owner;
    this.#balance = initialBalance;
  }

  deposit(amount) {
    if (amount <= 0) throw new Error("正の金額を入力してください");
    this.#balance += amount;
    return this;  // メソッドチェーン用
  }

  withdraw(amount) {
    if (amount > this.#balance) throw new Error("残高不足");
    this.#balance -= amount;
    return this;
  }

  get balance() { return this.#balance; }
  get owner()   { return this.#owner; }

  toString() {
    return `[${this.#owner}] 残高: ¥${this.#balance.toLocaleString()}`;
  }
}

const acc = new BankAccount("Alice", 10000);
acc.deposit(5000).deposit(3000); // メソッドチェーン
console.log(acc.balance); // → 18000（getter 経由）
console.log(acc.#balance); // → SyntaxError（クラス外からアクセス不可）
