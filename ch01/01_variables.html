② 宣言と変数

重要ポイント：変数とはデータを入れておく「箱」です。let キーワードで宣言します。
宣言した変数には =（代入演算子）で値を入れることができます。

変数の宣言と代入のイメージ

// メモリのイメージ

▼ 下のステップボタンを押して変数の変化を見てみましょう 
// let キーワードで変数を宣言する
let name;

// 値を代入していない変数は undefined になる
console.log(name); // → undefined
✅ let 変数名; で変数を宣言できます。値を代入する前は undefined です。
// 宣言と同時に値を代入する
let name = "Alice";

console.log(name); // → Alice
✅ let 変数名 = 値; で宣言と代入を同時に行えます。
let name = "Alice";
console.log(name); // → Alice

// 値を上書きできる（let は再代入可能）
name = "Bob";
console.log(name); // → Bob
✅ let で宣言した変数は後から値を変更できます。
// 複数の変数を宣言する
let name = "Alice";
let age  = 25;
let city = "Tokyo";

console.log(name); // → Alice
console.log(age);  // → 25
console.log(city); // → Tokyo
✅ 変数はいくつでも宣言できます。用途に合った名前をつけましょう。

③ let 以外の変数の宣言方法

重要ポイント：JavaScriptには let 以外に const と var があります。
現代のJavaScriptでは let と const を使うのが推奨です。

3つの宣言方法の比較（タブで切り替え）

let — 再代入可能な変数

let count = 0;
count = 1;      // OK：再代入できる
count = 2;      // OK：何度でも変更できる

// ブロックスコープ（{ } の中でのみ有効）
{
  let x = 10;
  console.log(x); // → 10
}
console.log(x); // → ReferenceError（ブロック外では使えない）

// 同じスコープで再宣言はできない
let name = "Alice";
let name = "Bob";   // → SyntaxError
✅ let の特徴：再代入OK・ブロックスコープ・再宣言NG
ループカウンタや後から変わる値に使います。

const — 再代入不可の定数

const PI = 3.14159;
PI = 3;  // → TypeError：再代入できない！

// 宣言と同時に代入が必須
const name;  // → SyntaxError：初期値が必要

// ただし、オブジェクト・配列の中身は変更できる
const user = { name: "Alice", age: 25 };
user.name = "Bob";   // OK：プロパティの変更はできる
user = {};             // → TypeError：変数自体への再代入はNG

// ブロックスコープ（let と同じ）
{
  const x = 10;
}
console.log(x); // → ReferenceError
✅ const の特徴：再代入NG・ブロックスコープ・宣言時に初期値必須
変わらない値（定数・設定値・関数など）に使います。基本的に const を使い、必要な時だけ let を使うのが現代のベストプラクティスです。

var — 旧来の宣言方法（非推奨）

// var は関数スコープ（ブロックスコープではない）
{
  var x = 10;
}
console.log(x); // → 10（ブロック外でも使える！）

// 再宣言できてしまう（バグの原因になりやすい）
var name = "Alice";
var name = "Bob";   // エラーにならない（危険）

// 巻き上げ（ホイスティング）が起きる
console.log(y); // → undefined（エラーにならない！）
var y = 5;
// ↑ 宣言が先頭に「巻き上げ」られるため
⚠️ var は現代のJavaScriptでは非推奨です。
関数スコープ・再宣言可能・ホイスティングなどの特性がバグを生みやすいため、 新しいコードでは let / const を使いましょう。
ただし試験では var の動作を問われることがあるため、動作の理解は必要です。

let / const / var の比較

項目letconstvar
再代入✅ できる❌ できない✅ できる
再宣言❌ できない❌ できない✅ できる
スコープブロックブロック関数
ホイスティングあり（TDZ）あり（TDZ）あり（undefined）
グローバル汚染なしなしあり（window.変数）
推奨度⭐⭐⭐⭐⭐⭐⭐（最推奨）❌ 非推奨
💡 TDZ（Temporal Dead Zone）とは？
let / const は宣言前にアクセスすると ReferenceError になります。 これを「一時的なデッドゾーン」と呼びます。 var のように undefined にはなりません。
