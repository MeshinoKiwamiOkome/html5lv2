② Web Storage（localStorage / sessionStorage）

重要ポイント：localStorage はブラウザを閉じても保持され、 sessionStorage はタブを閉じると消えます。 どちらも文字列のみ保存できます（オブジェクトは JSON.stringify が必要）。

localStorage vs sessionStorage 比較

特徴localStoragesessionStorage
データの保持 ブラウザを閉じても保持 タブ/ウィンドウを閉じると消える
スコープ 同一オリジン全体 同一タブのみ
容量 約 5MB 約 5MB
データ型 文字列のみ 文字列のみ
用途 設定・ログイン状態の保持 フォームの一時保存・セッション管理

🎮 Web Storage ライブデモ

// localStorage を操作してみよう（ページを再読み込みしてもデータが残る）
現在の localStorage の内容：
← setItem でデータを保存してみましょう
// localStorage の基本操作
localStorage.setItem("username", "Alice");
localStorage.getItem("username");   // → "Alice"
localStorage.removeItem("username");
localStorage.clear();               // 全削除
localStorage.length;               // → 保存件数
localStorage.key(0);               // → 0番目のキー名

// オブジェクトを保存する場合は JSON.stringify が必要
const user = { name: "Alice", age: 25 };
localStorage.setItem("user", JSON.stringify(user));

// 読み込み時は JSON.parse
const saved = JSON.parse(localStorage.getItem("user"));
console.log(saved.name); // → "Alice"

// sessionStorage も同じ API（スコープが異なるだけ）
sessionStorage.setItem("temp", "一時データ");

③ Canvas API

重要ポイント：<canvas> 要素と getContext("2d") で取得した描画コンテキストを使って 図形・テキスト・画像などを描画できます。

🎨 Canvas ライブ描画デモ

色： 
// Canvas の基本
const canvas = document.getElementById("myCanvas");
const ctx    = canvas.getContext("2d"); // 2D 描画コンテキスト

// 矩形
ctx.fillStyle   = "#4299e1";
ctx.fillRect(10, 10, 100, 60);   // x, y, width, height
ctx.strokeStyle = "#fff";
ctx.lineWidth   = 2;
ctx.strokeRect(10, 10, 100, 60);

// 円（パス）
ctx.beginPath();
ctx.arc(200, 50, 40, 0, Math.PI * 2); // x, y, r, 開始角, 終了角
ctx.fillStyle = "#68d391";
ctx.fill();

// テキスト
ctx.font      = "bold 24px sans-serif";
ctx.fillStyle = "#fff";
ctx.fillText("Hello Canvas!", 10, 120);

// グラデーション
const grad = ctx.createLinearGradient(0, 0, 200, 0);
grad.addColorStop(0,   "#4299e1");
grad.addColorStop(1,   "#9f7aea");
ctx.fillStyle = grad;
ctx.fillRect(0, 150, 200, 40);

④ Geolocation・History API・その他

Geolocation API

// 現在位置を1回取得
navigator.geolocation.getCurrentPosition(
  (position) => {
    const { latitude, longitude, accuracy } = position.coords;
    console.log(`緯度: ${latitude}`);
    console.log(`経度: ${longitude}`);
    console.log(`精度: ${accuracy}m`);
  },
  (error) => {
    console.log("エラー:", error.message);
  },
  { enableHighAccuracy: true, timeout: 5000 }
);

// 位置の継続監視
const watchId = navigator.geolocation.watchPosition(
  (pos) => console.log(pos.coords.latitude)
);
// 監視を停止
navigator.geolocation.clearWatch(watchId);

History API（SPA ルーティング）

// pushState：URL を変更してもページ遷移しない
history.pushState(
  { page: "about" },  // state オブジェクト
  "About",           // タイトル（多くのブラウザで無視）
  "/about"           // 新しい URL
);

// replaceState：現在の履歴エントリを置き換え
history.replaceState({ page: "home" }, "", "/");

// 前後の履歴に移動
history.back();     // 戻る
history.forward(); // 進む
history.go(-2);    // 2つ戻る

// popstate：ブラウザの戻る/進むボタンで発火
window.addEventListener("popstate", (e) => {
  console.log("URL が変わった:", location.pathname);
  console.log("state:", e.state);
  // ここでコンテンツを切り替える（SPA）
});

Web Workers

Web Workers とは：メインスレッドとは別のスレッドで JavaScript を実行する仕組みです。 重い計算処理をバックグラウンドで行い、UI のフリーズを防ぎます。 
// メインスレッド（main.js）
const worker = new Worker("worker.js");

// Worker にメッセージを送る
worker.postMessage({ type: "calculate", data: 1000000 });

// Worker からのメッセージを受け取る
worker.addEventListener("message", (e) => {
  console.log("結果:", e.data);
  worker.terminate(); // Worker を終了
});

// ---- worker.js ----
self.addEventListener("message", (e) => {
  const n = e.data.data;
  let sum = 0;
  for (let i = 1; i <= n; i++) sum += i;
  self.postMessage(sum); // 結果を返す
});

// Worker の制約
// ✅ XMLHttpRequest, fetch が使える
// ✅ setTimeout, setInterval が使える
// ❌ DOM にアクセスできない
// ❌ window オブジェクトにアクセスできない
Blob URL を使ったインライン Worker デモ

Intersection Observer API

Intersection Observer とは：要素がビューポートに入ったり 出たりするタイミングを効率的に監視する API です。 スクロールイベントより高パフォーマンスです。 
// Intersection Observer の基本
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        // 要素がビューポートに入った
        entry.target.classList.add("visible");
        observer.unobserve(entry.target); // 1回だけ
      }
    });
  },
  {
    threshold:  0.1,  // 10% 見えたら発火
    rootMargin: "0px"
  }
);

// 監視する要素を登録
document.querySelectorAll(".lazy-item")
  .forEach(el => observer.observe(el));

// 用途
// ✅ 遅延読み込み（lazy loading）
// ✅ スクロールアニメーション
// ✅ 無限スクロール
// ✅ 広告の表示計測

📌 まとめ

API主な用途重要ポイント
localStorage 永続的なデータ保存 文字列のみ・JSON.stringify が必要・約5MB
sessionStorage タブ内の一時データ タブを閉じると消える・同一タブのみ

WebSocket リアルタイム双方向通信 ws:// または wss:// プロトコル
