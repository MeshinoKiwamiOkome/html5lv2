1. Web Storage（localStorage / sessionStorage）

Web Storage はキー・バリュー形式でデータをブラウザに保存する仕組みです。localStorage と sessionStorage の2種類があり、永続性とスコープが異なります。

比較項目localStoragesessionStorage
永続性明示的に削除するまで永続タブ（セッション）を閉じると消える
スコープ同一オリジンの全タブ・ウィンドウで共有同一タブ内のみ（タブごとに独立）
容量約 5MB（ブラウザ依存）約 5MB（ブラウザ依存）
データ型文字列のみ（JSON.stringify で変換）文字列のみ（JSON.stringify で変換）
storage イベント他タブへ変更通知あり他タブへの通知なし
サーバー送信自動送信なし（Cookie と異なる）自動送信なし
Web Storage の CRUD 操作 
// ── localStorage ──

// 保存（setItem）
localStorage.setItem('username', 'Alice');
localStorage.setItem('count', String(42));          // 数値は文字列に変換
localStorage.setItem('user', JSON.stringify({ id:1, name:'Alice' }));

// 取得（getItem）
const name  = localStorage.getItem('username');    // 'Alice'
const count = parseInt(localStorage.getItem('count')); // 42（数値に変換）
const user  = JSON.parse(localStorage.getItem('user'));  // オブジェクトに変換
const none  = localStorage.getItem('missing');     // null（存在しない場合）

// 削除（removeItem）
localStorage.removeItem('username');

// 全削除（clear）
localStorage.clear();

// キー数とキー名
const len = localStorage.length;            // 保存件数
const key = localStorage.key(0);            // インデックスでキー名取得

// storage イベント（他タブからの変更を検知）
window.addEventListener('storage', (e) => {
  console.log(e.key);        // 変更されたキー
  console.log(e.oldValue);   // 変更前の値
  console.log(e.newValue);   // 変更後の値
  console.log(e.storageArea); // localStorage or sessionStorage
});

// sessionStorage も同じ API
sessionStorage.setItem('temp', 'session data');
📌 試験頻出：Web Storage の特性
  • 保存できるのは文字列のみ。オブジェクトは JSON.stringify() で変換が必要
  • 存在しないキーの getItem() は null を返す（undefined ではない）
  • localStorage は同一オリジンの全タブで共有される
  • sessionStorage はタブごとに独立。同じオリジンでも別タブは別ストレージ
  • Cookie と異なり、サーバーに自動送信されない
  • storage イベントは変更を行ったタブ自身には発火しない（他タブのみ）

2. IndexedDB

IndexedDB はブラウザ内の非同期トランザクション型データベースです。大量のデータや構造化データを保存でき、インデックスによる高速検索が可能です。

IndexedDB の基本操作 
// ── DB を開く（なければ作成）──
const request = indexedDB.open('MyDatabase', 1); // (DB名, バージョン)

// バージョンが上がったとき（スキーマ変更）
request.onupgradeneeded = (event) => {
  const db = event.target.result;

  // オブジェクトストアの作成
  const store = db.createObjectStore('users', {
    keyPath: 'id',          // プライマリキーのフィールド名
    autoIncrement: true    // 自動採番
  });

  // インデックスの作成
  store.createIndex('name', 'name', { unique: false });
  store.createIndex('email', 'email', { unique: true });
};

request.onsuccess = (event) => {
  const db = event.target.result;

  // ── データの追加（readwrite トランザクション）──
  const tx    = db.transaction('users', 'readwrite');
  const store = tx.objectStore('users');

  const addReq = store.add({ name: 'Alice', email: 'alice@example.com' });
  addReq.onsuccess = (e) => console.log('追加成功: id=', e.target.result);

  // ── データの取得（readonly トランザクション）──
  const tx2    = db.transaction('users', 'readonly');
  const store2 = tx2.objectStore('users');

  store2.get(1).onsuccess = (e) => console.log(e.target.result);

  // ── 全件取得 ──
  store2.getAll().onsuccess = (e) => console.log(e.target.result);

  // ── 更新 ──
  store.put({ id: 1, name: 'Alice Updated', email: 'alice@example.com' });

  // ── 削除 ──
  store.delete(1);

  // トランザクション完了
  tx.oncomplete = () => console.log('トランザクション完了');
  tx.onerror    = (e) => console.log('エラー:', e.target.error);
};

request.onerror = (event) => {
  console.log('DB オープンエラー:', event.target.error);
};
📌 試験頻出：IndexedDB の仕組み
  • すべての操作は非同期（コールバック or Promise ベース）
  • データの読み書きは必ずトランザクション内で行う
  • トランザクションモード: 'readonly'（読み取り専用）/ 'readwrite'（読み書き）
  • onupgradeneeded はバージョンが上がったときのみ呼ばれる（スキーマ変更の場所）
  • キーパス（keyPath）でオブジェクトのプロパティをプライマリキーに指定できる
  • Web Storage より大容量（数十〜数百MB）のデータを扱える

3. File API

File API を使うと、ユーザーが選択したファイルを JavaScript で読み込めます。FileReader オブジェクトが中心的な役割を担います。

FileReader の使い方 
// input[type=file] からファイルを取得
const input = document.getElementById('fileInput');
input.addEventListener('change', (e) => {
  const file = e.target.files[0]; // File オブジェクト

  // File オブジェクトのプロパティ
  console.log(file.name);         // ファイル名
  console.log(file.size);         // バイト数
  console.log(file.type);         // MIME タイプ（例: 'text/plain'）
  console.log(file.lastModified); // 最終更新日時（Unix タイムスタンプ ms）

  const reader = new FileReader();

  // ── テキストとして読み込む ──
  reader.readAsText(file, 'UTF-8');
  reader.onload = (e) => {
    console.log(e.target.result); // 文字列
  };

  // ── Data URL として読み込む（画像プレビュー等）──
  reader.readAsDataURL(file);
  reader.onload = (e) => {
    // 'data:image/png;base64,iVBORw0KGgo...'
    imgEl.src = e.target.result;
  };

  // ── ArrayBuffer として読み込む（バイナリ処理）──
  reader.readAsArrayBuffer(file);
  reader.onload = (e) => {
    const buffer = e.target.result; // ArrayBuffer
    const view   = new Uint8Array(buffer);
    console.log(view[0], view[1]); // 最初の2バイト
  };

  // イベント一覧
  reader.onloadstart  = () => console.log('読み込み開始');
  reader.onprogress   = (e) => console.log(`${e.loaded} / ${e.total}`);
  reader.onloadend    = () => console.log('読み込み終了');
  reader.onerror      = () => console.log('エラー');
  reader.onabort      = () => console.log('中断');

  // 読み込み中断
  reader.abort();
});
メソッド結果（result プロパティ）用途
readAsText(file, encoding)文字列（DOMString）テキストファイルの読み込み
readAsDataURL(file)Data URL（base64 エンコード）画像プレビュー・埋め込み
readAsArrayBuffer(file)ArrayBuffer（バイナリ）バイナリ処理・音声/画像加工
readAsBinaryString(file)バイナリ文字列（非推奨）レガシー用途（ArrayBuffer 推奨）
abort()—読み込みを中断
⚠️ FileReader は非同期

readAsText() などを呼んだ直後に reader.result を参照しても null です。必ず onload イベント内で結果を取得してください。また、同じ FileReader インスタンスで複数のファイルを同時に読み込もうとすると InvalidStateError が発生します。

4. バイナリデータ（ArrayBuffer・TypedArray）

ArrayBuffer は固定長のバイナリデータバッファです。直接アクセスはできず、TypedArray（型付き配列）または DataView を通じて操作します。

ArrayBuffer と TypedArray の操作 
// ── ArrayBuffer の作成 ──
const buffer = new ArrayBuffer(16); // 16バイトのバッファ
console.log(buffer.byteLength);     // → 16

// ── TypedArray でビューを作成 ──
const uint8  = new Uint8Array(buffer);   // 1バイト符号なし整数 × 16要素
const int16  = new Int16Array(buffer);   // 2バイト符号あり整数 × 8要素
const float32= new Float32Array(buffer); // 4バイト浮動小数点 × 4要素

// 書き込み
uint8[0] = 255;   // 0xFF
uint8[1] = 128;   // 0x80
uint8[2] = 256;   // オーバーフロー → 0（Uint8Array はクランプしない）

// 同じバッファを別の型で見ると値が変わる
console.log(uint8[0]);   // → 255
console.log(int16[0]);   // → -128 (リトルエンディアン: 0xFF80 = -128)

// ── Uint8ClampedArray（Canvas で使用）──
const clamped = new Uint8ClampedArray(4);
clamped[0] = 300; // → 255 にクランプ（オーバーフローしない）
clamped[1] = -5;  // → 0 にクランプ

// ── DataView（エンディアン制御が必要な場合）──
const view = new DataView(buffer);
view.setUint8(0, 0xFF);
view.setInt16(2, 1000, true);  // true = リトルエンディアン
const val = view.getInt16(2, true); // → 1000

// ── スライス（部分コピー）──
const slice = buffer.slice(0, 8);    // 先頭8バイトをコピー
const sub   = uint8.subarray(0, 4); // コピーなし（同じバッファを参照）
TypedArrayバイト数/要素値の範囲説明
Int8Array1-128 〜 127符号あり8ビット整数
Uint8Array10 〜 255符号なし8ビット整数
Uint8ClampedArray10 〜 255（クランプ）Canvas ImageData で使用
Int16Array2-32768 〜 32767符号あり16ビット整数
Uint16Array20 〜 65535符号なし16ビット整数
Int32Array4-2^31 〜 2^31-1符号あり32ビット整数
Uint32Array40 〜 2^32-1符号なし32ビット整数
Float32Array4IEEE 754 単精度Web Audio API で使用
Float64Array8IEEE 754 倍精度高精度数値計算
📌 試験頻出：Uint8Array vs Uint8ClampedArray
  • Uint8Array: 256 を代入すると 0 になる（ビットマスク: 256 & 0xFF = 0）
  • Uint8ClampedArray: 256 を代入すると 255 になる（クランプ）、-1 は 0 になる
  • Canvas の getImageData() が返す ImageData.data は Uint8ClampedArray
  • Web Audio API の getByteTimeDomainData() は Uint8Array を使用

QUESTION 1 / 6
localStorage と sessionStorage の違いについて正しい説明はどれですか？
✅ 正解：B
B ○ localStorage のデータは明示的に削除するまで永続します。ブラウザを閉じても消えません。
A ✗ 全タブで共有されるのは localStorage です。sessionStorage はタブごとに独立しています。
C ✗ Web Storage はサーバーに自動送信されません（Cookie と異なる重要な特性）。
D ✗ 両者の容量は同程度（約5MB）です。
QUESTION 2 / 6
IndexedDB でデータを追加する際に必要なトランザクションモードはどれですか？
✅ 正解：C（'readwrite'）
IndexedDB のトランザクションモードは2種類です：
• 'readonly' — 読み取り専用（get・getAll など）
• 'readwrite' — 読み書き両方（add・put・delete など）
'writeonly' や 'write' というモードは存在しません。データの追加・更新・削除には必ず 'readwrite' を指定します。
QUESTION 3 / 6
FileReader で画像ファイルを読み込んで <img> タグの src に直接セットしたい場合、どのメソッドを使うべきですか？
✅ 正解：B（readAsDataURL()）
readAsDataURL() は data:image/png;base64,... 形式の Data URL を生成します。この文字列をそのまま img.src に代入すると画像が表示されます。
• readAsText(): テキストファイル向け（バイナリを文字列として読むと文字化け）
• readAsArrayBuffer(): バイナリ処理向け（img.src には使えない）
• readAsBinaryString(): 非推奨。ArrayBuffer を使うべき
QUESTION 4 / 6
以下のコードを実行したとき、result の値はいくつになりますか？
const arr = new Uint8Array(1); arr[0] = 300; const result = arr[0];
✅ 正解：C（44）
Uint8Array は 0〜255 の範囲の符号なし8ビット整数です。300 を代入すると ビットマスク（モジュロ演算）が行われます：
300 % 256 = 44
（300 = 0x12C → 下位8ビット = 0x2C = 44）

※ Uint8ClampedArray の場合は 255 にクランプされます。
この違いが試験で問われることがあります：
• Uint8Array: オーバーフロー → ビットマスク（300 → 44）
• Uint8ClampedArray: オーバーフロー → クランプ（300 → 255）
QUESTION 5 / 6
IndexedDB の onupgradeneeded イベントが発火するタイミングとして正しいものをすべて選んでください。
※ 正しいものをすべてクリックして「確認」ボタンを押してください
✅ 正解：A・C
onupgradeneeded は以下の場合に発火します：
A ○ DB が存在しない（初回作成）→ バージョン 0 から指定バージョンへのアップグレード
B ✗ 毎回発火しません。バージョンが変わったときのみです
C ○ open() のバージョン引数が現在の DB バージョンより大きい場合
D ✗ データ操作では発火しません。スキーマ変更専用のイベントです

onupgradeneeded 内でのみ createObjectStore() や createIndex() が呼べます。
