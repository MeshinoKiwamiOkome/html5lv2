1. Geolocation API

Geolocation API はデバイスの位置情報（緯度・経度・高度など）を取得します。navigator.geolocation オブジェクトを通じてアクセスし、ユーザーの許可が必要です。

Geolocation API の基本 
// Geolocation API のサポート確認
if ('geolocation' in navigator) {
  console.log('Geolocation API 利用可能');
}

// ── 1回だけ位置を取得 ──
navigator.geolocation.getCurrentPosition(
  // 成功コールバック
  (position) => {
    const coords = position.coords;
    console.log(coords.latitude);         // 緯度（-90〜90）
    console.log(coords.longitude);        // 経度（-180〜180）
    console.log(coords.altitude);         // 高度（m）null の場合あり
    console.log(coords.accuracy);         // 水平精度（m）
    console.log(coords.altitudeAccuracy);  // 高度精度（m）null の場合あり
    console.log(coords.heading);          // 方位角（0〜360）null の場合あり
    console.log(coords.speed);            // 速度（m/s）null の場合あり
    console.log(position.timestamp);      // 取得時刻（Unix ms）
  },
  // エラーコールバック
  (error) => {
    switch (error.code) {
      case 1: console.log('PERMISSION_DENIED — ユーザーが拒否');    break;
      case 2: console.log('POSITION_UNAVAILABLE — 位置情報取得不可'); break;
      case 3: console.log('TIMEOUT — タイムアウト');                 break;
    }
  },
  // オプション
  {
    enableHighAccuracy: true,  // GPS 等の高精度センサーを使用（バッテリー消費増）
    timeout:            5000,  // タイムアウト時間（ms）
    maximumAge:         0      // キャッシュ許容時間（0=常に新規取得）
  }
);

// ── 継続的に位置を監視 ──
const watchId = navigator.geolocation.watchPosition(
  (position) => {
    console.log('位置更新:', position.coords.latitude, position.coords.longitude);
  },
  (error) => console.log('エラー:', error.code),
  { enableHighAccuracy: true, timeout: 10000, maximumAge: 1000 }
);

// 監視を停止
navigator.geolocation.clearWatch(watchId); // watchId を渡す
プロパティ型説明null の可能性
coords.latitudenumber緯度（度）-90〜90なし
coords.longitudenumber経度（度）-180〜180なし
coords.accuracynumber水平精度（メートル）なし
coords.altitudenumber高度（メートル）あり
coords.altitudeAccuracynumber高度精度（メートル）あり
coords.headingnumber方位角（度）0=北、90=東あり
coords.speednumber速度（m/s）あり
position.timestampnumber取得時刻（Unix ミリ秒）なし
📌 試験頻出：Geolocation API のポイント
  • エラーコード: 1=PERMISSION_DENIED / 2=POSITION_UNAVAILABLE / 3=TIMEOUT
  • watchPosition() の戻り値は watchId（数値）。clearWatch(watchId) で監視停止
  • enableHighAccuracy: true にすると GPS 等を使い精度が上がるが、バッテリー消費が増える
  • maximumAge はキャッシュされた位置情報の許容時間（ms）。0 は常に新規取得
  • HTTPS または localhost でのみ動作（セキュリティ要件）

2. DeviceOrientation イベント

DeviceOrientation イベントはデバイスの物理的な向き（傾き）をリアルタイムで通知します。alpha・beta・gamma の3軸で表現されます。

DeviceOrientation イベントの使い方 
// DeviceOrientation イベントの登録
window.addEventListener('deviceorientation', (event) => {
  // alpha: Z軸周りの回転（コンパス方位）0〜360°
  //   0 = 北、90 = 西、180 = 南、270 = 東
  console.log('alpha:', event.alpha);

  // beta: X軸周りの回転（前後の傾き）-180〜180°
  //   0 = 水平、90 = 画面が上向き、-90 = 画面が下向き
  console.log('beta:',  event.beta);

  // gamma: Y軸周りの回転（左右の傾き）-90〜90°
  //   0 = 垂直、90 = 右に傾く、-90 = 左に傾く
  console.log('gamma:', event.gamma);

  // absolute: 地球の座標系に対する絶対値かどうか
  console.log('absolute:', event.absolute);
}, true);

// iOS 13+ では許可が必要
if (typeof DeviceOrientationEvent.requestPermission === 'function') {
  DeviceOrientationEvent.requestPermission()
    .then(state => {
      if (state === 'granted') {
        window.addEventListener('deviceorientation', handler);
      }
    });
}

// CSS transform で傾きを可視化
function handleOrientation(e) {
  const el = document.getElementById('device');
  el.style.transform =
    `rotateZ(${e.alpha}deg) ` +
    `rotateX(${e.beta}deg) `  +
    `rotateY(${e.gamma}deg)`;
}
プロパティ軸範囲説明
alphaZ軸（垂直軸）0〜360°コンパス方位。0=北、90=西、180=南、270=東
betaX軸（横軸）-180〜180°前後の傾き。0=水平、90=画面が上向き
gammaY軸（縦軸）-90〜90°左右の傾き。正=右傾き、負=左傾き
absolute—boolean地球の座標系に対する絶対値かどうか
📌 試験頻出：alpha・beta・gamma の意味
  • alpha（Z軸）: デバイスを水平に置いたときのコンパス方位。0〜360°
  • beta（X軸）: 前後の傾き。スマホを立てた状態が 90°（または -90°）
  • gamma（Y軸）: 左右の傾き。-90〜90° の範囲
  • iOS 13 以降は DeviceOrientationEvent.requestPermission() でユーザー許可が必要

3. DeviceMotion イベント

DeviceMotion イベントの使い方 
// DeviceMotion イベントの登録
window.addEventListener('devicemotion', (event) => {

  // ── 加速度（重力を除く）──
  const acc = event.acceleration;
  if (acc) {
    console.log('x:', acc.x); // 左右（右が正）m/s²
    console.log('y:', acc.y); // 前後（前が正）m/s²
    console.log('z:', acc.z); // 上下（上が正）m/s²
  }

  // ── 加速度（重力を含む）──
  const accG = event.accelerationIncludingGravity;
  if (accG) {
    // 静止時: z ≈ -9.8（重力加速度）
    console.log('z(重力込み):', accG.z);
  }

  // ── 回転レート ──
  const rot = event.rotationRate;
  if (rot) {
    console.log('alpha:', rot.alpha); // Z軸周りの回転速度（°/s）
    console.log('beta:',  rot.beta);  // X軸周りの回転速度（°/s）
    console.log('gamma:', rot.gamma); // Y軸周りの回転速度（°/s）
  }

  // ── イベント間隔 ──
  console.log('interval:', event.interval); // ms（通常 16〜50ms）
});

// ── シェイク検出の例 ──
let lastShake = 0;
window.addEventListener('devicemotion', (e) => {
  const acc = e.accelerationIncludingGravity;
  if (!acc) return;
  const total = Math.abs(acc.x) + Math.abs(acc.y) + Math.abs(acc.z);
  if (total > 30) {  // 閾値
    const now = Date.now();
    if (now - lastShake > 1000) {
      console.log('シェイク検出!');
      lastShake = now;
    }
  }
});
💡 DeviceOrientation と DeviceMotion の違い
  • DeviceOrientation: デバイスの向き（角度）を提供。alpha/beta/gamma（度）
  • DeviceMotion: デバイスの動き（加速度・回転速度）を提供。m/s² や °/s
  • DeviceOrientation は「今どの向きか」、DeviceMotion は「どれだけ速く動いているか」

🧩 演習問題

4-5 デバイスアクセスの理解を確認しましょう。

QUESTION 1 / 6
Geolocation API でユーザーが位置情報の使用を拒否したとき、エラーコールバックに渡される error.code の値はどれですか？
✅ 正解：A（1 = PERMISSION_DENIED）
Geolocation API のエラーコードは3種類です：
• 1: PERMISSION_DENIED — ユーザーが位置情報の使用を拒否した
• 2: POSITION_UNAVAILABLE — ネットワーク障害・GPS 信号なしなどで位置情報を取得できない
• 3: TIMEOUT — timeout オプションで指定した時間内に取得できなかった
0（UNKNOWN_ERROR）は仕様上存在しません。
QUESTION 2 / 6
watchPosition() の戻り値として正しいものはどれですか？
✅ 正解：B（監視 ID = 数値）
watchPosition() は数値の watchId を返します。この ID を clearWatch(watchId) に渡すことで監視を停止できます。
getCurrentPosition() は戻り値なし（undefined）です。
位置情報は成功コールバックの引数として渡される GeolocationPosition オブジェクトで取得します。
QUESTION 3 / 6
DeviceOrientation イベントの gamma プロパティが表す動きはどれですか？
✅ 正解：C（Y軸周りの回転 = 左右の傾き）
DeviceOrientation の3軸：
• alpha（Z軸）: コンパス方位。0〜360°。0=北
• beta（X軸）: 前後の傾き。-180〜180°
• gamma（Y軸）: 左右の傾き。-90〜90°。正=右傾き
加速度は DeviceMotion イベントの acceleration プロパティで取得します。
QUESTION 4 / 6
Geolocation API の enableHighAccuracy: true オプションについて正しい説明はどれですか？
✅ 正解：B
enableHighAccuracy: true は GPS などの高精度センサーの使用を要求しますが：
• デバイスが GPS をサポートしていない場合は効果がない
• バッテリー消費が増加し、取得に時間がかかる場合がある
• 必ずしも高精度が保証されるわけではない
false（デフォルト）の場合は Wi-Fi・IP アドレスなど低精度だが高速な方法を使います。
QUESTION 5 / 6（複数選択）
Geolocation API の getCurrentPosition() の第3引数（オプション）に指定できるプロパティをすべて選んでください。
※ 正しいものをすべてクリックして「確認」を押してください
✅ 正解：A・B・D
PositionOptions オブジェクトに指定できるプロパティは3つです：
A ○ enableHighAccuracy: 高精度センサーを使用するか（boolean）
B ○ timeout: タイムアウト時間（ms）。超過すると TIMEOUT エラー
C ✗ accuracy: そのようなオプションは存在しない
D ○ maximumAge: キャッシュされた位置情報の許容時間（ms）。0=常に新規取得
E ✗ interval: そのようなオプションは存在しない（watchPosition の更新間隔はブラウザ依存）
QUESTION 6 / 6（穴埋め）
以下のコードの空欄を埋めてください。位置監視を開始し、後で停止します。
// 位置監視を開始（戻り値を保存） const watchId = navigator.geolocation.[___A___]( (pos) => console.log(pos.coords.latitude), (err) => console.log(err.code), { enableHighAccuracy: true } ); // 監視を停止 navigator.geolocation.[___B___](watchId);
