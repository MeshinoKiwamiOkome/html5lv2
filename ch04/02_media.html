1. HTMLMediaElement の基礎

<audio> と <video> は共通の HTMLMediaElement インターフェースを継承しています。JavaScript からメソッド・プロパティ・イベントを通じて完全に制御できます。

audio / video 要素の基本 HTML 
<!-- video 要素 -->
<video
  id="myVideo"
  width="640"
  height="360"
  controls              <!-- ブラウザ標準コントロール表示 -->
  autoplay             <!-- 自動再生（muted と併用が必要なことが多い）-->
  loop                 <!-- ループ再生 -->
  muted                <!-- ミュート -->
  preload="metadata"  <!-- none / metadata / auto -->
  poster="thumb.jpg"  <!-- サムネイル画像 -->
>
  <source src="video.mp4"  type="video/mp4">
  <source src="video.webm" type="video/webm">
  <!-- フォールバックテキスト -->
  お使いのブラウザは video 要素をサポートしていません。
</video>

<!-- audio 要素 -->
<audio id="myAudio" controls preload="auto">
  <source src="audio.ogg" type="audio/ogg">
  <source src="audio.mp3" type="audio/mpeg">
</audio>
プロパティ / メソッド型説明
play()Promise再生開始。Promise を返す（ES2016以降）
pause()void一時停止
load()voidメディアをリセットして再読み込み
currentTimenumber現在の再生位置（秒）。書き込みでシーク
durationnumberメディアの総再生時間（秒）
pausedboolean一時停止中かどうか
endedboolean再生終了したかどうか
volumenumber音量（0.0〜1.0）
mutedbooleanミュート状態
playbackRatenumber再生速度（1.0が標準、0.5で半速、2.0で倍速）
readyStatenumberメディアの準備状態（0〜4）
networkStatenumberネットワーク状態（0〜3）
bufferedTimeRangesバッファリング済みの時間範囲
canPlayType(type)string再生可能かを返す（''/'maybe'/'probably'）
📌 試験頻出：readyState の値
  • HAVE_NOTHING (0) — メディアデータなし
  • HAVE_METADATA (1) — メタデータ（duration等）のみ取得済み
  • HAVE_CURRENT_DATA (2) — 現在位置のデータあり（次フレームなし）
  • HAVE_FUTURE_DATA (3) — 現在位置と次フレームのデータあり
  • HAVE_ENOUGH_DATA (4) — 最後まで再生できるデータあり

2. メディアイベント

主要メディアイベントの登録 
const video = document.getElementById('myVideo');

// 再生・停止系
video.addEventListener('play',    () => console.log('再生開始'));
video.addEventListener('pause',   () => console.log('一時停止'));
video.addEventListener('ended',   () => console.log('再生終了'));
video.addEventListener('playing', () => console.log('再生中（バッファ後）'));

// 時間・進行系
video.addEventListener('timeupdate', () => {
  console.log(`現在位置: ${video.currentTime.toFixed(2)}秒`);
});
video.addEventListener('seeking',  () => console.log('シーク中'));
video.addEventListener('seeked',   () => console.log('シーク完了'));

// 読み込み系
video.addEventListener('loadstart',    () => console.log('読み込み開始'));
video.addEventListener('loadedmetadata',() => {
  console.log(`duration: ${video.duration}秒`);
});
video.addEventListener('loadeddata',    () => console.log('データ読み込み完了'));
video.addEventListener('canplay',       () => console.log('再生可能'));
video.addEventListener('canplaythrough',() => console.log('最後まで再生可能'));
video.addEventListener('waiting',       () => console.log('バッファ待ち'));
video.addEventListener('stalled',       () => console.log('データ取得停滞'));

// エラー
video.addEventListener('error', (e) => {
  console.log(`エラー: ${video.error.code}`); // 1〜4
});
イベント名発火タイミング
loadstartメディアの読み込みが開始されたとき
loadedmetadataduration・dimensions などのメタデータが読み込まれたとき
loadeddata現在の再生位置のデータが読み込まれたとき
canplay再生可能になったとき（バッファ不足で止まる可能性あり）
canplaythrough最後まで止まらずに再生できると判断されたとき
playplay() が呼ばれたとき / autoplay 開始時
playing実際に再生が始まったとき（バッファ後も含む）
pausepause() が呼ばれたとき
timeupdatecurrentTime が変化したとき（約250ms間隔）
seekingシーク操作が始まったとき
seekedシーク操作が完了したとき
ended再生が終了したとき
volumechangevolume または muted が変化したとき
ratechangeplaybackRate が変化したとき
waiting次のフレームが利用可能になるまで待機中
errorエラーが発生したとき

3. Web Audio API

Web Audio API は高度な音声処理を JavaScript で実現します。AudioContext を中心に、複数の AudioNode をグラフ状に接続して処理パイプラインを構築します。

Web Audio API の基本構造 
// AudioContext の作成
const audioCtx = new AudioContext();

// ── 音声ソース → 処理ノード → 出力先 ──
// OscillatorNode（音声生成）
const osc = audioCtx.createOscillator();
osc.type      = 'sine';    // sine / square / sawtooth / triangle
osc.frequency.setValueAtTime(440, audioCtx.currentTime); // 440Hz = ラ

// GainNode（音量調整）
const gainNode = audioCtx.createGain();
gainNode.gain.setValueAtTime(0.5, audioCtx.currentTime);

// AnalyserNode（波形分析）
const analyser = audioCtx.createAnalyser();
analyser.fftSize = 256;  // FFT サイズ（2の累乗）
const bufLen = analyser.frequencyBinCount; // fftSize / 2
const dataArr = new Uint8Array(bufLen);

// ノードを接続: osc → gain → analyser → destination
osc.connect(gainNode);
gainNode.connect(analyser);
analyser.connect(audioCtx.destination); // スピーカー出力

// 再生開始・停止
osc.start(audioCtx.currentTime);
osc.stop(audioCtx.currentTime + 2); // 2秒後に停止

// 波形データの取得（アニメーションループ内で呼ぶ）
analyser.getByteTimeDomainData(dataArr);  // 時間領域（波形）
analyser.getByteFrequencyData(dataArr);   // 周波数領域（スペクトラム）
📌 試験頻出：AudioContext のノード種類
  • ソースノード：OscillatorNode・AudioBufferSourceNode・MediaElementAudioSourceNode
  • 処理ノード：GainNode（音量）・BiquadFilterNode（フィルター）・DelayNode（遅延）・AnalyserNode（分析）
  • 出力先：AudioContext.destination（スピーカー）
  • ノードは connect() で接続し、グラフ構造を形成する
  • AudioContext はブラウザのユーザー操作後に作成する（自動再生ポリシー）

3 Web Audio API ビジュアライザー

OscillatorNode で音を生成し、AnalyserNode で波形・周波数をリアルタイム可視化します。

▶ 「音声開始」をクリックすると OscillatorNode が起動します（音が出ます）
⚠️ 自動再生ポリシーに注意

多くのブラウザでは、ユーザー操作なしに AudioContext を起動すると suspended 状態になります。必ずボタンクリックなどのユーザー操作をトリガーにして audioCtx.resume() を呼ぶか、操作後に AudioContext を作成してください。

4. TextTrack API（字幕・トラック）

track 要素と TextTrack API 
<!-- HTML: track 要素で字幕を指定 -->
<video id="v">
  <source src="video.mp4" type="video/mp4">
  <track
    kind="subtitles"  <!-- subtitles / captions / descriptions / chapters / metadata -->
    src="sub_ja.vtt"
    srclang="ja"
    label="日本語"
    default
  >
</video>

// JavaScript から TextTrack を操作
const video = document.getElementById('v');
const tracks = video.textTracks;          // TextTrackList
const track  = tracks[0];                 // TextTrack

// トラックの表示モード
track.mode = 'showing';  // showing / hidden / disabled

// キュー（字幕の1行）へのアクセス
const cues = track.cues;  // TextTrackCueList
const cue  = cues[0];    // VTTCue
console.log(cue.startTime); // 開始時刻（秒）
console.log(cue.endTime);   // 終了時刻（秒）
console.log(cue.text);      // 字幕テキスト

// cuechange イベント（字幕が切り替わるたびに発火）
track.addEventListener('cuechange', () => {
  const activeCues = track.activeCues;
  if (activeCues.length > 0) {
    console.log(activeCues[0].text);
  }
});

// WebVTT フォーマット（.vtt ファイル）
// WEBVTT
//
// 00:00:01.000 --> 00:00:04.000
// こんにちは、HTML5 の世界へようこそ。
//
// 00:00:05.000 --> 00:00:08.000
// Web API を学びましょう。

5. canPlayType() によるメディア形式判定

canPlayType() の使い方 
const video = document.createElement('video');
const audio = document.createElement('audio');

// 戻り値: '' (再生不可) / 'maybe' (おそらく可) / 'probably' (ほぼ確実に可)

// 動画形式
video.canPlayType('video/mp4');
video.canPlayType('video/mp4; codecs="avc1.42E01E, mp4a.40.2"');
video.canPlayType('video/webm');
video.canPlayType('video/webm; codecs="vp9, vorbis"');
video.canPlayType('video/ogg; codecs="theora"');

// 音声形式
audio.canPlayType('audio/mpeg');          // MP3
audio.canPlayType('audio/ogg; codecs="vorbis"');
audio.canPlayType('audio/wav');
audio.canPlayType('audio/aac');

// 判定ロジックの例
function canPlay(el, type) {
  const result = el.canPlayType(type);
  if (result === 'probably') return '✅ probably';
  if (result === 'maybe')    return '⚠️ maybe';
  return '❌ 再生不可';
}

// 注意: 'probably' でも実際に再生できない場合がある
// 'maybe' はコーデック情報なしで判定できない場合に返る

5 canPlayType() ブラウザ対応チェッカー

このブラウザが各メディア形式を再生できるかをリアルタイムで判定します。

「判定を実行」ボタンを押してください
💡 canPlayType() の戻り値の意味
  • ''（空文字）— 再生不可。このブラウザはこの形式をサポートしていない
  • 'maybe' — コーデック情報なしでは確定できないが、再生できる可能性がある
  • 'probably' — コーデックまで指定されており、再生できる可能性が高い
  • コーデック情報（codecs="..."）を含めると 'maybe' → 'probably' になることが多い

🧩 演習問題

4-2 メディア要素の API の理解を確認しましょう。

QUESTION 1 / 5
HTMLMediaElement の readyState が HAVE_METADATA (1) のとき、取得できる情報として正しいものはどれですか？
✅ 正解：B
HAVE_METADATA (1) は「メタデータが読み込まれた状態」です。この状態では duration（総再生時間）・video の width/height・textTracks などのメタデータにアクセスできます。
• readyState 0（HAVE_NOTHING）: データなし
• readyState 1（HAVE_METADATA）: メタデータのみ
• readyState 2（HAVE_CURRENT_DATA）: 現在フレームのデータあり
• readyState 3（HAVE_FUTURE_DATA）: 次フレームもあり
• readyState 4（HAVE_ENOUGH_DATA）: 最後まで再生可能
QUESTION 2 / 5
メディアの再生位置が変化するたびに約250ms間隔で発火するイベントはどれですか？
✅ 正解：C（timeupdate）
timeupdate イベントは currentTime が変化するたびに発火します。発火頻度はブラウザ依存ですが、仕様では約250ms間隔が推奨されています。
• playing: 実際の再生が始まったとき（バッファ後も含む）
• progress: データのダウンロードが進行中のとき
• currentchange: そのようなイベントは存在しない
QUESTION 3 / 5
Web Audio API で音声の音量を調整するために使用するノードはどれですか？
✅ 正解：B（GainNode）
GainNode は音量（ゲイン）を調整するノードです。gainNode.gain.value または gainNode.gain.setValueAtTime() で音量を設定します。
• AnalyserNode: 波形・周波数データの分析（視覚化に使用）
• OscillatorNode: 音声信号の生成（サイン波・矩形波など）
• DelayNode: 音声の遅延処理
QUESTION 4 / 5
track 要素の kind 属性に指定できる値として存在しないものはどれですか？
✅ 正解（存在しない値）：C（transcript）
track 要素の kind 属性に指定できる値は以下の5種類です：
• subtitles: 字幕（翻訳）
• captions: キャプション（音声説明を含む字幕）
• descriptions: 映像の音声説明
• chapters: チャプターナビゲーション
• metadata: スクリプトから使用するメタデータ
transcript は存在しません。

const video = document.getElementById('myVideo'); // 再生速度を2倍に video.[___A___] = 2; // 音量を50%に（0.0〜1.0 の範囲） video.[___B___] = 0.5;
