1. Canvas 要素と 2D コンテキスト

Canvas 要素はピクセルベースの描画領域を提供します。JavaScript から getContext('2d') で 2D 描画コンテキストを取得し、図形・テキスト・画像を描画します。

Canvas 基本セットアップ 
// HTML: <canvas id="myCanvas" width="400" height="300"></canvas>

const canvas = document.getElementById('myCanvas');
const ctx    = canvas.getContext('2d');  // 2D コンテキスト取得

// 矩形の描画
ctx.fillStyle   = '#553c9a';           // 塗りつぶし色
ctx.fillRect(50, 50, 100, 80);         // fillRect(x, y, width, height)

// 枠線のみの矩形
ctx.strokeStyle = '#9f7aea';
ctx.lineWidth   = 3;
ctx.strokeRect(200, 50, 100, 80);      // strokeRect(x, y, width, height)

// 矩形の消去
ctx.clearRect(0, 0, canvas.width, canvas.height); // 全体クリア
メソッド / プロパティ説明引数
getContext('2d')2D 描画コンテキストを取得—
fillRect(x,y,w,h)塗りつぶし矩形を描画x, y, 幅, 高さ
strokeRect(x,y,w,h)枠線矩形を描画x, y, 幅, 高さ
clearRect(x,y,w,h)指定領域をクリア（透明化）x, y, 幅, 高さ
fillStyle塗りつぶし色・グラデーション色文字列 / CanvasGradient
strokeStyle線の色・グラデーション色文字列 / CanvasGradient
lineWidth線の太さ（px）数値
📌 試験頻出：getContext() の引数
  • '2d' → 2D 描画コンテキスト（CanvasRenderingContext2D）
  • 'webgl' → WebGL コンテキスト（3D描画）
  • コンテキストが取得できない場合は null が返る
  • Canvas 要素は CSS でサイズを変えると描画がぼやける。width/height 属性で論理サイズを指定すること

パス（Path）による図形描画

パス・円・テキスト描画 
// ── パスによる三角形 ──
ctx.beginPath();            // パス開始（必須）
ctx.moveTo(100, 20);         // 開始点へ移動
ctx.lineTo(180, 160);        // 直線を引く
ctx.lineTo(20,  160);
ctx.closePath();            // パスを閉じる（開始点に戻る）
ctx.fillStyle = 'tomato';
ctx.fill();                  // 塗りつぶし
ctx.stroke();               // 枠線

// ── 円弧 ──
ctx.beginPath();
ctx.arc(
  250, 90,   // 中心 x, y
  60,         // 半径
  0,          // 開始角度（ラジアン）
  Math.PI * 2, // 終了角度（2π = 360°）
  false       // 反時計回り？
);
ctx.fillStyle = '#9f7aea';
ctx.fill();

// ── テキスト ──
ctx.font      = 'bold 24px sans-serif';
ctx.fillStyle = '#2d3748';
ctx.fillText('Hello Canvas', 50, 220);  // fillText(text, x, y)
ctx.strokeText('Outline', 200, 220);    // strokeText(text, x, y)

2. 状態の保存と復元（save / restore）

Canvas の描画状態（色・線幅・変形行列・クリッピング等）はスタック構造で管理されます。save() で現在の状態をスタックに積み、restore() で取り出します。

save() / restore() の使い方 
// 初期状態: fillStyle = 'black'
ctx.fillStyle = 'black';

ctx.save();               // ① 状態をスタックに保存
  ctx.fillStyle = 'red';
  ctx.save();             // ② さらに保存
    ctx.fillStyle = 'blue';
    ctx.fillRect(10, 10, 50, 50); // blue で描画
  ctx.restore();          // ② を復元 → fillStyle = 'red'
  ctx.fillRect(70, 10, 50, 50); // red で描画
ctx.restore();           // ① を復元 → fillStyle = 'black'
ctx.fillRect(130, 10, 50, 50); // black で描画

// 保存される状態：fillStyle, strokeStyle, lineWidth,
//   font, globalAlpha, 変形行列, クリッピング領域 など
⚠️ save/restore を使わないと状態が汚染される

変形（translate/rotate/scale）を行う際は必ず save() → 変形 → 描画 → restore() のパターンを使います。restore() を忘れると、以降の描画がすべて変形された状態で行われます。

3. 座標変換（translate・rotate・scale）

変形メソッドの使い方 
// ── translate（平行移動）──
ctx.save();
ctx.translate(100, 100);   // 原点を (100,100) に移動
ctx.fillRect(-25, -25, 50, 50); // 新原点を中心に描画
ctx.restore();

// ── rotate（回転）──
ctx.save();
ctx.translate(200, 100);   // 回転の中心へ移動
ctx.rotate(Math.PI / 4);   // 45° 回転（ラジアン指定）
ctx.fillRect(-25, -25, 50, 50);
ctx.restore();

// ── scale（拡大縮小）──
ctx.save();
ctx.translate(300, 100);
ctx.scale(2, 0.5);          // x方向2倍, y方向0.5倍
ctx.fillRect(-25, -25, 50, 50);
ctx.restore();

// ── 度数をラジアンに変換 ──
const toRad = deg => deg * Math.PI / 180;

4. requestAnimationFrame によるアニメーション

requestAnimationFrame（rAF）はブラウザの描画タイミングに同期してコールバックを呼び出します。setInterval より滑らかで、タブが非アクティブのときは自動的に停止します。

requestAnimationFrame の基本パターン 
let animId = null;  // アニメーション ID
let x = 0;

function animate(timestamp) {  // timestamp: 経過時間(ms)
  ctx.clearRect(0, 0, canvas.width, canvas.height);

  // 描画処理
  ctx.fillStyle = '#553c9a';
  ctx.fillRect(x, 100, 40, 40);

  x += 2;
  if (x > canvas.width) x = -40;

  animId = requestAnimationFrame(animate); // 次フレームを予約
}

// 開始
animId = requestAnimationFrame(animate);

// 停止
cancelAnimationFrame(animId);

// setInterval との違い
// setInterval  → 指定間隔で強制実行（60fps保証なし・タブ非表示でも動く）
// rAF          → ブラウザ描画に同期（通常60fps・タブ非表示で自動停止）
比較項目requestAnimationFramesetInterval
フレームレートディスプレイのリフレッシュレートに同期（通常60fps）指定間隔（精度低め）
タブ非表示時自動的に停止（省電力）動き続ける（CPU消費）
コールバック引数DOMHighResTimeStamp（経過時間）なし
キャンセルcancelAnimationFrame(id)clearInterval(id)
用途アニメーション・ゲーム定期的なデータ更新
📌 試験頻出：rAF の特性
  • コールバックには DOMHighResTimeStamp（ミリ秒精度の経過時間）が渡される
  • タブが非アクティブになると自動停止 → バッテリー・CPU の節約
  • 戻り値（ID）を cancelAnimationFrame() に渡すと停止できる
  • 1フレームあたり約 16.7ms（60fps の場合）

5. イメージデータとピクセル操作

ImageData によるピクセル操作 
// ImageData の取得
const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
const data = imageData.data;  // Uint8ClampedArray [R,G,B,A, R,G,B,A, ...]

// ピクセル (x, y) のインデックス計算
const getIdx = (x, y) => (y * canvas.width + x) * 4;

// グレースケール変換
for (let i = 0; i < data.length; i += 4) {
  const r = data[i];
  const g = data[i + 1];
  const b = data[i + 2];
  // data[i + 3] はアルファ値（0〜255）
  const gray = r * 0.299 + g * 0.587 + b * 0.114;
  data[i] = data[i + 1] = data[i + 2] = gray;
}

// 変更を Canvas に書き戻す
ctx.putImageData(imageData, 0, 0);

// 新しい ImageData を作成
const newData = ctx.createImageData(100, 100); // 100x100 の空データ
💡 Uint8ClampedArray の特性

各チャンネル（R/G/B/A）は 0〜255 の整数値。255 を超えると自動的に 255 にクランプされ、0 未満は 0 になります（通常の Uint8Array と異なり、オーバーフローしない）。

6. SVG（Scalable Vector Graphics）

SVG はベクター形式の画像フォーマットで、XML ベースで記述します。Canvas（ラスター）と異なり、どんなサイズに拡大しても品質が劣化しません。

SVG の基本構文 
<!-- インライン SVG -->
<svg width="400" height="200" xmlns="http://www.w3.org/2000/svg">

  <!-- 矩形 -->
  <rect x="10" y="10" width="80" height="60"
        fill="#553c9a" stroke="#9f7aea" stroke-width="2"/>

  <!-- 円 -->
  <circle cx="160" cy="50" r="40" fill="tomato"/>

  <!-- 楕円 -->
  <ellipse cx="260" cy="50" rx="60" ry="30" fill="#48bb78"/>

  <!-- 直線 -->
  <line x1="10" y1="120" x2="390" y2="120"
        stroke="#4299e1" stroke-width="3"/>

  <!-- パス -->
  <path d="M 10 180 L 100 140 L 190 180 Z"
        fill="#ed8936" opacity="0.8"/>

  <!-- テキスト -->
  <text x="220" y="175" font-size="18" fill="#2d3748">SVG Text</text>

</svg>

📌 試験頻出：SVG と Canvas の使い分け
  • SVG は DOM の一部 → CSS スタイリング・JavaScript イベントが使える
  • Canvas は描画後にピクセルデータのみ残る → 個々の図形を後から操作できない
  • SVG は拡大・縮小しても品質が落ちない（ベクター）
  • Canvas は大量の図形を高速に描画するのに向いている（ゲーム等）

4-1 グラフィックス・アニメーションの理解を確認しましょう。

QUESTION 1 / 6
Canvas の 2D 描画コンテキストを取得する正しいコードはどれですか？
✅ 正解：B
canvas.getContext('2d') が正しい書き方です。引数に '2d' を渡すと CanvasRenderingContext2D オブジェクトが返ります。WebGL を使う場合は 'webgl' または 'webgl2' を指定します。コンテキストが取得できない場合は null が返るため、実際のコードでは null チェックが推奨されます。
QUESTION 2 / 6
以下のコードを実行した結果、最後の fillRect で使われる fillStyle の色はどれですか？
ctx.fillStyle = 'black'; ctx.save(); ctx.fillStyle = 'red'; ctx.save(); ctx.fillStyle = 'blue'; ctx.restore(); ctx.fillRect(0, 0, 50, 50); // ← この fillRect の色は？ ctx.restore();
✅ 正解：B（red）
スタックの動きを追うと：
① save() → スタック: [black]
② fillStyle = 'red'
③ save() → スタック: [black, red]
④ fillStyle = 'blue'
⑤ restore() → スタック: [black]、fillStyle = 'red' に戻る
⑥ fillRect → red で描画
save/restore はスタック（LIFO）構造で動作します。
QUESTION 3 / 6
requestAnimationFrame について正しい説明をすべて選んでください。（複数選択）
※ 正しいものをすべてクリックして「確認」ボタンを押してください
✅ 正解：A・C・E
A ○ コールバックの第1引数は DOMHighResTimeStamp（経過時間 ms）です。
B ✗ 「常に高速」ではありません。ディスプレイのリフレッシュレート（通常60fps）に同期するため、それ以上速くはなりません。
C ○ タブが非表示になると自動停止し、省電力になります。
D ✗ 戻り値はアニメーション ID（数値）で、cancelAnimationFrame() に渡して停止できます。
E ○ cancelAnimationFrame(id) でアニメーションを停止できます。
QUESTION 4 / 6
Canvas の arc() メソッドで完全な円を描くとき、終了角度として正しいのはどれですか？
✅ 正解：B（Math.PI * 2）
arc() の角度はラジアン（radian）で指定します。
• 360° = 2π ≈ 6.283 → Math.PI * 2
• 180° = π → Math.PI
• 90° = π/2 → Math.PI / 2
度数をラジアンに変換するには deg * Math.PI / 180 を使います。360（度数）を直接指定しても正しく動作しません。
QUESTION 5 / 6
SVG と Canvas の違いについて、誤っている説明はどれですか？
✅ 正解（誤り）：C
「Canvas の方が SVG より常にパフォーマンスが高い」は誤りです。
• 要素数が少ない場合は SVG の方が効率的なこともあります
• Canvas が有利なのは、大量の図形を高速に再描画する場合（ゲーム・リアルタイム処理）
• SVG は DOM ベースのため、少数の要素のインタラクションには向いています
A・B・D はすべて正しい説明です。
