① BEM 命名規則

重要ポイント：BEM（Block・Element・Modifier）は CSS クラス名の命名規則です。 block__element--modifier の形式で コンポーネントの構造を表現します。
Block
独立したコンポーネント。それ自体で意味を持つ。
例：card・button・nav・header
Element
Block の一部。Block なしでは意味を持たない。
記法：block__element（アンダースコア2つ）
例：card__title・card__image・nav__item
Modifier
Block または Element のバリエーション・状態。
記法：block--modifier（ハイフン2つ）
例：button--primary・card--featured・nav__item--active 
/* BEM の実践例：カードコンポーネント */

/* HTML */
<div class="card card--featured">
  <img  class="card__image" src="..." alt="...">
  <div  class="card__body">
    <h2 class="card__title">タイトル</h2>
    <p  class="card__text">本文</p>
    <a  class="card__btn card__btn--primary">詳しく見る</a>
    <a  class="card__btn card__btn--secondary">保存</a>
  </div>
</div>

/* CSS */
.card               { border-radius: 8px; overflow: hidden; }
.card--featured     { border: 2px solid #553c9a; }
.card__image        { width: 100%; }
.card__body         { padding: 16px; }
.card__title        { font-size: 1.2em; }
.card__btn          { padding: 8px 16px; }
.card__btn--primary { background: #553c9a; color: #fff; }
.card__btn--secondary{ background: transparent; }

/* BEM のメリット */
/* ✅ クラス名だけで構造が分かる */
/* ✅ 詳細度が低く（クラス1つ）管理しやすい */
/* ✅ コンポーネントの再利用性が高い */
/* ❌ クラス名が長くなりがち */

② CSS アーキテクチャ

モダンな CSS リセット

/* モダンな CSS リセット（最小限版）*/
*,
*::before,
*::after {
  box-sizing: border-box;
}

* {
  margin:  0;
  padding: 0;
}

html {
  font-size:           16px;
  -webkit-text-size-adjust: 100%;
}

body {
  line-height: 1.5;
  -webkit-font-smoothing:  antialiased;
}

img, picture, video, canvas, svg {
  display:   block;
  max-width: 100%;
}

input, button, textarea, select {
  font: inherit;
}

p, h1, h2, h3, h4, h5, h6 {
  overflow-wrap: break-word;
}

#root, #__next {
  isolation: isolate;
}

詳細度を低く保つ設計

/* ❌ 詳細度が高すぎる（上書きが困難）*/
#header .nav ul li a { color: blue; }
/* 詳細度 (1,1,3) → 上書きするには同等以上が必要 */

/* ✅ BEM で詳細度を低く保つ */
.nav__link { color: blue; }
/* 詳細度 (0,1,0) → 簡単に上書き可能 */

/* :where() で詳細度をゼロにする */
:where(h1, h2, h3) { font-weight: bold; }
/* 詳細度 (0,0,0) → 何でも上書き可能 */

/* :is() は引数の最高詳細度を持つ */
:is(#id, .class, p) { color: red; }
/* 詳細度 (1,0,0) ← #id の詳細度が適用 */

/* !important の乱用を避ける */
/* ユーティリティクラスには許容される場合も */
.sr-only {
  position: absolute !important;
  width:    1px !important;
  height:   1px !important;
  overflow: hidden !important;
  clip:     rect(0,0,0,0) !important;
}

@layer（カスケードレイヤー）

@layer とは：CSS のカスケード順序を明示的に制御できる機能です。 同じ詳細度でも、後のレイヤーが優先されます。 
/* @layer でカスケード順序を定義 */
@layer reset, base, components, utilities;
/* 後に定義したレイヤーが優先される */
/* utilities > components > base > reset */

@layer reset {
  * { box-sizing: border-box; margin: 0; }
}

@layer base {
  body { font-family: sans-serif; }
  a    { color: blue; }
}

@layer components {
  .btn { padding: 8px 16px; }
}

@layer utilities {
  .mt-4 { margin-top: 1rem; }
  .text-center { text-align: center; }
}

/* レイヤー外のスタイルは最優先 */
.override { color: red; } /* @layer より優先 */

CSS パフォーマンス最適化

/* 1. セレクタの最適化 */
/* ❌ 右から左に評価されるため非効率 */
div ul li a span { color: red; }

/* ✅ クラスで直接指定 */
.nav-link-text { color: red; }

/* 2. contain プロパティ */
.widget {
  contain: layout style paint;
  /* ブラウザに独立したコンテキストを伝える */
}

/* 3. content-visibility */
.below-fold {
  content-visibility: auto;
  /* 画面外の要素のレンダリングをスキップ */
}

/* 4. GPU アクセラレーション */
.animated {
  transform:  translateZ(0); /* GPU レイヤー化 */
  will-change: transform;
}

/* 5. font-display で FOIT を防ぐ */
@font-face {
  font-display: swap;
}

/* 6. 不要なアニメーションを避ける */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}

③ 試験頻出パターン

🎯 詳細度の計算 超頻出
#id .class p → (1,1,1)
.a:hover::before → (0,2,1)
クラスを何個重ねても ID に勝てない
:where() は詳細度ゼロ
📦 box-sizing の計算 超頻出
content-box：実幅 = width + padding×2 + border×2
border-box：実幅 = width（そのまま）
モダン開発では border-box が推奨
↔️ Flexbox の軸 超頻出
justify-content → 主軸方向
align-items → 交差軸方向
column のとき主軸は縦
flex: 1 = grow:1 shrink:1 basis:0%
⊞ Grid の span 超頻出
grid-column: 2 / span 3 → 2列目から3列分
grid-column: 1 / -1 → 全列にまたがる
fr は残りスペースの比率単位
📌 position の基準 超頻出
absolute → 最近傍の positioned 祖先
fixed → ビューポート（transform 例外あり）
sticky → top 等の閾値が必須
static では z-index 無効
📱 メディアクエリ 頻出
モバイルファースト → min-width
デスクトップファースト → max-width
rem は html 基準・em は親基準
clamp(min, val, max) でメディアクエリ不要
✨ transition vs animation 頻出
transition → トリガー（:hover等）が必要
animation → 自動再生・@keyframes 必要
fill-mode: forwards → 最終フレーム維持
transform は GPU 処理で高速
🔧 CSS 変数 頻出
--name: value で定義
var(--name, fallback) で参照
子要素は親の変数を継承する
JS: el.style.setProperty('--name', val)
🎨 色の記法 標準
HSL: H=色相(0-360°) S=彩度(%) L=明度(%)
rgba(r,g,b,a) のアルファは 0〜1
currentColor は現在の color 値を参照
oklch は知覚的に均一なモダン記法
🔤 タイポグラフィ 標準
font-weight: bold = 700
line-height は単位なし推奨
text-overflow: ellipsis には
white-space: nowrap + overflow: hidden も必要
