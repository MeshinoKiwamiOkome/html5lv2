📋 配列の基本

配列はインデックス（0始まり）で要素にアクセスするオブジェクトです。

// 配列リテラル
const fruits = ['apple', 'banana', 'cherry'];
console.log(fruits[0]);      // "apple"
console.log(fruits.length);  // 3
console.log(fruits.at(-1));   // "cherry"（末尾要素、ES2022）

// 多次元配列
const matrix = [[1,2],[3,4],[5,6]];
console.log(matrix[1][0]);   // 3

// Array コンストラクタ
const arr1 = new Array(3);       // [empty × 3]（長さ3の空配列）
const arr2 = Array.from({length:3}, (_,i) => i); // [0, 1, 2]

スプレッド構文と分割代入

// スプレッド構文：配列を展開
const a = [1, 2, 3];
const b = [4, 5, 6];
const merged = [...a, ...b];    // [1, 2, 3, 4, 5, 6]
const copy   = [...a];           // シャローコピー

// 分割代入（Destructuring）
const [first, second, ...rest] = [10, 20, 30, 40];
console.log(first);  // 10
console.log(second); // 20
console.log(rest);   // [30, 40]

// デフォルト値付き分割代入
const [x = 0, y = 0, z = 99] = [1, 2];
console.log(z); // 99（未定義なのでデフォルト値）

// 変数の入れ替え
let p = 1, q = 2;
[p, q] = [q, p];
console.log(p, q); // 2 1

🔧 配列メソッド一覧

追加・削除（破壊的メソッド）

メソッド説明戻り値破壊的
push(...items)末尾に追加新しい length✅
pop()末尾を削除削除した要素✅
unshift(...items)先頭に追加新しい length✅
shift()先頭を削除削除した要素✅
splice(start, del, ...items)任意位置で削除・挿入削除した要素の配列✅
sort(fn)並び替え元の配列✅
reverse()逆順元の配列✅
fill(val, start, end)要素を埋める元の配列✅

変換・検索（非破壊的メソッド）

メソッド説明戻り値破壊的
map(fn)各要素を変換新しい配列❌
filter(fn)条件に合う要素を抽出新しい配列❌
reduce(fn, init)累積処理累積値❌
find(fn)条件に合う最初の要素要素 or undefined❌
findIndex(fn)条件に合う最初のインデックスインデックス or -1❌
indexOf(val)値の最初のインデックスインデックス or -1❌
includes(val)値が含まれるかboolean❌
some(fn)1つでも条件を満たすかboolean❌
every(fn)すべて条件を満たすかboolean❌
slice(start, end)部分配列を取得新しい配列❌
concat(...arr)配列を結合新しい配列❌
join(sep)文字列に変換文字列❌
flat(depth)ネストを平坦化新しい配列❌
flatMap(fn)map後にflat(1)新しい配列❌
forEach(fn)各要素を処理（戻り値なし）undefined❌
 
const nums = [1, 2, 3, 4, 5];

// map：各要素を2倍に変換
console.log(nums.map(n => n * 2));         // [2, 4, 6, 8, 10]

// filter：偶数のみ抽出
console.log(nums.filter(n => n % 2 === 0)); // [2, 4]

// reduce：合計を計算
console.log(nums.reduce((acc, n) => acc + n, 0)); // 15

// find：最初に3より大きい要素
console.log(nums.find(n => n > 3));          // 4

// some / every
console.log(nums.some(n => n > 4));          // true（5が4より大きい）
console.log(nums.every(n > 0));             // true（全て0より大きい）

// flat：ネスト配列を平坦化
console.log([1,[2,[3]]].flat());            // [1, 2, [3]]（深さ1）
console.log([1,[2,[3]]].flat(Infinity));   // [1, 2, 3]（全て展開）

// sort：数値ソートは比較関数が必要
console.log([10,9,2].sort());              // [10, 2, 9]（文字列比較！）
console.log([10,9,2].sort((a,b) => a-b)); // [2, 9, 10]（数値昇順）
⚠️ 試験頻出：sort() の落とし穴
引数なしの sort() は要素を文字列として比較します。 数値を正しくソートするには sort((a, b) => a - b)（昇順）または sort((a, b) => b - a)（降順）と比較関数を渡す必要があります。

✏️ 演習問題

Q1 配列メソッドの戻り値について空欄を埋めてください。（穴埋め）

push() の戻り値は配列の新しい である。 pop() の戻り値は した要素である。 forEach() の戻り値は である。 filter() の戻り値は条件を満たす要素からなる である。

Q2 次のコードの実行結果として正しいものはどれですか？（選択）
const result = [1, 2, 3, 4, 5] .filter(n => n % 2 !== 0) .reduce((acc, n) => acc + n, 0); console.log(result);
Q3 破壊的メソッド（元の配列を変更する）をすべて選んでください。（複数選択）
Q4 偶数のみを取り出して2倍にし、合計を求めるコードを完成させてください。（コード補完）
const nums = [1, 2, 3, 4, 5, 6]; const result = nums .(n => n % 2 === 0) .(n => n * 2) .((acc, n) => acc + n, 0); console.log(result); // 24
