① viewport メタタグ

重要ポイント：レスポンシブデザインの第一歩は <meta name="viewport"> タグの設定です。 これがないとモバイルブラウザがページをデスクトップ幅で表示してしまいます。 
<!-- viewport メタタグ（必須）-->
<meta name="viewport"
      content="width=device-width, initial-scale=1.0">

<!-- 各パラメータの意味 -->
<!-- width=device-width：デバイスの実際の幅に合わせる -->
<!-- initial-scale=1.0：初期ズーム倍率 1.0（等倍）-->
<!-- user-scalable=no：ユーザーのズームを禁止（非推奨）-->
<!-- maximum-scale=1.0：最大ズーム倍率（アクセシビリティ上非推奨）-->

<!-- viewport-fit=cover：ノッチ対応 -->
<meta name="viewport"
      content="width=device-width, initial-scale=1.0, viewport-fit=cover">
⚠️ viewport タグなしの問題： モバイルブラウザはデフォルトで 980px 幅でページを表示し、 それを縮小して表示します。 width=device-width を指定することで デバイスの実際の幅でレンダリングされます。

② メディアクエリ

重要ポイント：@media ルールで画面幅などの条件に応じて スタイルを切り替えます。 min-width（モバイルファースト）と max-width（デスクトップファースト）の 使い分けが重要です。

メディアクエリの構文

/* 基本構文 */
@media メディアタイプ and (条件) {
  /* 条件に合致したときのスタイル */
}

/* モバイルファースト（min-width）← 推奨 */
/* デフォルト：モバイル向けスタイル */
.container { padding: 16px; }

@media (min-width: 768px) {
  .container { padding: 24px; } /* タブレット以上 */
}
@media (min-width: 1024px) {
  .container { padding: 32px; } /* PC以上 */
}

/* デスクトップファースト（max-width）*/
.container { padding: 32px; }

@media (max-width: 1023px) {
  .container { padding: 24px; }
}
@media (max-width: 767px) {
  .container { padding: 16px; }
}

/* 範囲指定（CSS Media Queries Level 4）*/
@media (768px <= width < 1024px) {
  /* タブレットのみ */
}

/* 複数条件の組み合わせ */
@media (min-width: 768px) and (max-width: 1023px) {
  /* タブレットのみ */
}
@media (min-width: 768px), (orientation: landscape) {
  /* タブレット以上 OR 横向き */
}

特性例説明
width min-width: 768px ビューポートの幅
height min-height: 600px ビューポートの高さ
orientation orientation: landscape 横向き / 縦向き

④ 相対単位

重要ポイント：レスポンシブデザインでは絶対単位（px）より 相対単位を活用します。 rem はルート要素基準、em は親要素基準、 vw/vh はビューポート基準です。

rem と em の違い

/* rem：ルート要素（html）の font-size を基準 */
html { font-size: 16px; } /* ← 基準 */
.box { padding: 1rem; }  /* = 16px（常に html 基準）*/
.box { font-size: 1.5rem; } /* = 24px */

/* em：親要素の font-size を基準 */
.parent { font-size: 20px; }
.child  { font-size: 1.5em; } /* = 30px（20px × 1.5）*/
.grandchild { font-size: 1.5em; } /* = 45px（30px × 1.5）*/
/* em は入れ子で複利的に大きくなる点に注意！ */

/* 実践的な使い方 */
html { font-size: 62.5%; } /* 16px × 62.5% = 10px */
/* → 1rem = 10px になり計算しやすい */
body { font-size: 1.6rem; } /* = 16px */
h1   { font-size: 3.2rem; } /* = 32px */

vw / vh / vmin / vmax

/* vw：ビューポート幅の 1% */
.hero { width: 100vw; } /* 画面幅いっぱい */
.title{ font-size: 5vw; }  /* 画面幅の5%のフォントサイズ */

/* vh：ビューポート高さの 1% */
.full-screen { height: 100vh; } /* 画面高さいっぱい */
.half-screen { height: 50vh; }

/* vmin：vw と vh の小さいほう */
/* vmax：vw と vh の大きいほう */
.square { width: 50vmin; height: 50vmin; }
/* 縦横どちらにも収まる正方形 */

/* dvh（Dynamic Viewport Height）*/
/* モバイルのアドレスバー表示/非表示を考慮 */
.mobile-full { height: 100dvh; }

/* vw + rem の組み合わせ（fluid typography）*/
h1 { font-size: calc(1.5rem + 2vw); }
/* 最小1.5rem、画面が広がるにつれて大きくなる */

clamp() - 流体タイポグラフィ

clamp(最小値, 推奨値, 最大値)： 値を最小値と最大値の間に収めます。 メディアクエリなしでレスポンシブなサイズを実現できます。 
/* clamp(最小値, 推奨値, 最大値) */
h1 {
  font-size: clamp(1.5rem, 4vw, 3rem);
  /* 最小1.5rem、推奨4vw、最大3rem */
  /* 画面幅に応じて自動的にスケール */
}

.container {
  width: clamp(320px, 90%, 1200px);
  /* 最小320px、推奨90%、最大1200px */
}

.card {
  padding: clamp(12px, 3vw, 32px);
}

/* min() / max() */
.box { width: min(100%, 600px); }
/* 100% と 600px の小さいほう */

.box { font-size: max(1rem, 2vw); }
/* 1rem と 2vw の大きいほう */

CSS 単位一覧

単位種類基準用途
px絶対1ピクセル border・shadow など精密な指定
rem相対html の font-size フォントサイズ・余白（推奨）
em相対親要素の font-size コンポーネント内の相対サイズ
%相対親要素のサイズ 幅・高さの割合指定
vw相対ビューポート幅の1% 全幅レイアウト・流体タイポグラフィ
vh相対ビューポート高さの1% 全画面セクション
dvh相対動的ビューポート高さ モバイルのアドレスバー対応
vmin相対vw/vh の小さいほう 正方形・アイコン
vmax相対vw/vh の大きいほう 背景要素
ch相対"0" 文字の幅 テキスト幅の制御
lh相対line-height 行高さ基準の余白

⑤ モバイルファーストとブレークポイント

/* モバイルファースト：小さい画面から書き始める */

/* ① ベーススタイル（モバイル）*/
.grid {
  display: grid;
  grid-template-columns: 1fr;       /* 1列 */
  gap: 16px;
}
.nav-links { display: none; }     /* ナビ非表示 */
.hamburger { display: block; }   /* ハンバーガー表示 */

/* ② タブレット（768px〜）*/
@media (min-width: 768px) {
  .grid {
    grid-template-columns: repeat(2, 1fr); /* 2列 */
  }
}

/* ③ デスクトップ（1024px〜）*/
@media (min-width: 1024px) {
  .grid {
    grid-template-columns: repeat(3, 1fr); /* 3列 */
  }
  .nav-links { display: flex; }   /* ナビ表示 */
  .hamburger { display: none; }   /* ハンバーガー非表示 */
}

/* ④ 大画面（1280px〜）*/
@media (min-width: 1280px) {
  .grid {
    grid-template-columns: repeat(4, 1fr); /* 4列 */
  }
}

Container Queries（コンテナクエリ）

/* Container Queries：コンテナのサイズに応じてスタイルを変える */
/* ビューポートではなく親要素のサイズを基準にできる */

.card-container {
  container-type: inline-size; /* コンテナとして登録 */
  container-name: card;
}

@container card (min-width: 400px) {
  .card {
    display: flex;
    flex-direction: row;
  }
}

📌 まとめ

技術説明重要ポイント
viewport メタタグ デバイス幅に合わせる width=device-width, initial-scale=1.0 が必須
@media (min-width) モバイルファースト 小さい画面から書き始める（推奨）
@media (max-width) デスクトップファースト 大きい画面から書き始める
rem html 基準の相対単位 フォント・余白に推奨
em 親要素基準の相対単位 入れ子で複利的に変化する点に注意
vw / vh ビューポート基準 全画面レイアウト・流体タイポグラフィ
clamp(min, pref, max) 範囲付き流体サイズ メディアクエリなしでレスポンシブ
@container コンテナクエリ 親要素サイズ基準（ビューポートではない）
