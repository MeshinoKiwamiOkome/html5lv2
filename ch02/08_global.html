🌐 グローバルオブジェクトと Call オブジェクト

JavaScriptには実行環境が自動的に生成する2種類の特殊なオブジェクトがあります。

オブジェクト生成タイミング役割ブラウザでの名前
グローバルオブジェクト スクリプト実行開始時 グローバル変数・関数を保持。トップレベルの this window / globalThis
Call オブジェクト
（実行コンテキスト）		 関数呼び出しごと ローカル変数・引数・arguments を保持 -（内部的な概念） 
// グローバルオブジェクト
console.log(typeof window);      // "object"（ブラウザ）
console.log(typeof globalThis);  // "object"（全環境共通・ES2020）

// varで宣言したグローバル変数はwindowのプロパティになる
var x = 42;
console.log(window.x); // 42

// let/constはwindowのプロパティにならない
let y = 10;
console.log(window.y); // undefined

// Call オブジェクト（実行コンテキスト）のイメージ
function outer(a) {       // Call オブジェクト①: { a, b, inner }
  let b = 2;
  function inner(c) {    // Call オブジェクト②: { c } + outer参照
    return a + b + c;  // スコープチェーンで a, b を探す
  }
  return inner(3);
}
console.log(outer(1)); // 6

🔭 スコープチェーンと名前解決

変数を参照するとき、JavaScriptは現在のスコープ → 外側のスコープ → ... → グローバルスコープ の順に変数を探します。この仕組みをスコープチェーンといいます。 

const x = 'global';       // グローバルスコープ

function outer() {
  const x = 'outer';     // outer のスコープ（グローバルを隠す）
  const y = 'from outer';

  function inner() {
    const z = 'from inner';
    console.log(x); // "outer"（outer のスコープで発見）
    console.log(y); // "from outer"（outer のスコープで発見）
    console.log(z); // "from inner"（自身のスコープで発見）
  }
  inner();
}
outer();

// with 文（非推奨・試験対策として把握）
// with はオブジェクトをスコープチェーンの先頭に追加する
// strict mode では SyntaxError になる
const obj = { a: 1, b: 2 };
// with (obj) { console.log(a + b); } // 3（非推奨）
⚠️ with 文について： with 文はオブジェクトのプロパティをスコープチェーンの先頭に追加しますが、 パフォーマンス低下・デバッグ困難・strict mode での禁止などの理由から非推奨です。 試験では「with 文は strict mode で使えない」という点を押さえましょう。

🔒 クロージャ（Closure）

クロージャとは、関数が定義されたスコープの変数を、 そのスコープが終了した後も参照し続けられる仕組みです。 

// クロージャの基本例
function makeCounter() {
  let count = 0;  // この変数がクロージャに閉じ込められる

  return {
    increment() { count++; },
    decrement() { count--; },
    getCount()  { return count; }
  };
}

const counter = makeCounter();
counter.increment();
counter.increment();
counter.increment();
counter.decrement();
console.log(counter.getCount()); // 2

// count は外から直接アクセスできない（プライベート変数）
console.log(counter.count); // undefined

// 独立したカウンター（それぞれ別のクロージャ）
const c1 = makeCounter();
const c2 = makeCounter();
c1.increment();
console.log(c1.getCount()); // 1
console.log(c2.getCount()); // 0（独立している）
💡 クロージャの用途
① プライベート変数の実現（外部から直接変更できない変数）
② 状態の保持（関数呼び出し間で値を維持）
③ カリー化・部分適用（関数を返す高階関数）
④ モジュールパターン（IIFE + クロージャでカプセル化）
⚠️ メモリリークに注意： クロージャが大きなオブジェクトへの参照を保持し続けると、 ガーベジコレクションが解放できずメモリリークの原因になります。 不要になったクロージャは参照を切りましょう。

// 0〜9 のランダム整数
const rand = Math.floor(Math.random() * 10);

// min〜max のランダム整数
function randInt(min, max) {
  return Math.floor(Math.random() * (max - min + 1)) + min;
}

// floor vs trunc（負の数で違いが出る）

const arr = [3, 1, 4, 1, 5, 9];
console.log(Math.max(...arr)); // 9
