📜 JavaScriptとECMAScriptの関係

JavaScript は Netscape 社が開発したスクリプト言語で、 現在は ECMAScript（ECMA-262） という標準仕様に基づいて実装されています。 「JavaScript」はブランド名、「ECMAScript」は仕様書の名前と覚えましょう。

💡 試験ポイント： JavaScriptの標準化を担う団体は ECMA International（欧州電子計算機工業会）です。 仕様書の正式名称は ECMA-262 といいます。

🏷 HTMLからJavaScriptを利用する方法

① インラインスクリプト

<script> タグの中に直接コードを書く方法です。

<!-- インラインスクリプト -->
<script>
  console.log('Hello, JavaScript!');
</script>

② 外部ファイルの読み込み

src 属性で外部の .js ファイルを指定します。外部ファイルを指定した場合、タグ内のコードは無視されます。

<!-- 外部ファイルの読み込み -->
<script src="./js/app.js"></script>

<!-- ❌ src指定時はタグ内コードが無視される -->
<script src="app.js">
  console.log('これは実行されない'); // 無視される
</script>

③ scriptタグの配置場所

配置場所特徴推奨度
<head> 内 HTML解析前に実行。DOMが存在しないためDOM操作不可 △（defer付きなら○）
</body> 直前 DOM構築後に実行。ページ表示が先に完了する ○

④ defer / async 属性

<!-- 通常：HTML解析をブロックして即実行 -->
<script src="app.js"></script>

<!-- defer：HTML解析と並行DL → 解析完了後に実行（順序保証） -->
<script src="app.js" defer></script>

<!-- async：HTML解析と並行DL → DL完了次第すぐ実行（順序不定） -->
<script src="app.js" async></script>
属性ダウンロード実行タイミング実行順序用途
なし HTML解析をブロック 即時 記述順 -
defer 並行（非同期） HTML解析完了後 記述順を保証 DOM操作スクリプト
async 並行（非同期） DL完了次第 不定 独立した分析ツール等
⚠️ 注意： defer と async は 外部ファイル（src属性あり）のみ有効です。 インラインスクリプトには効果がありません。

🔒 厳格モード（Strict Mode）

'use strict'; を記述すると 厳格モード が有効になります。 ES5 で導入され、バグの原因となりやすいコードをエラーとして検出します。 

// ファイル全体に適用（先頭に記述）
'use strict';

// 関数スコープのみに適用
function myFunc() {
  'use strict';
  // この関数内だけ厳格モード
}

// ES6モジュール（type="module"）は自動的に厳格モード
// <script type="module" src="app.js"></script>

厳格モードで禁止されること

禁止事項通常モード厳格モード
未宣言変数への代入グローバル変数として作成ReferenceError
with 文の使用動作するSyntaxError
同名の引数動作するSyntaxError
this が undefined（関数内）グローバルオブジェクトundefined
delete で変数削除失敗（エラーなし）SyntaxError
✅ 推奨： ES6 の type="module" を使うと自動的に厳格モードが適用されます。 また、クラス内のコードも常に厳格モードで動作します。
