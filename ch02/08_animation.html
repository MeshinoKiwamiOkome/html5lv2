① transition（トランジション）

重要ポイント：transition はプロパティの値が変化するときに アニメーションを付けます。 property duration timing-function delay の4値で指定します。 状態変化（:hover など）のトリガーが必要です。

🎮 transition プレイグラウンド

← コントロールを操作してコードを確認しましょう
/* transition の基本 */
.btn {
  background: #553c9a;
  transition: background 0.3s ease;
  /* property duration timing-function */
}
.btn:hover { background: #2c7a7b; }

/* 複数プロパティ */
.card {
  transition:
    transform  0.3s ease,
    box-shadow 0.3s ease,
    opacity    0.2s ease;
}

/* all で全プロパティ（パフォーマンスに注意）*/
.box { transition: all 0.3s ease; }

/* delay：遅延してから開始 */
.delayed {
  transition: opacity 0.5s ease 0.2s;
  /* property duration timing delay */
}

/* transition できないプロパティ */
/* display, visibility（none→block は不可）*/
/* → opacity + pointer-events で代替 */
.fade-out {
  opacity:         0;
  pointer-events: none;
  transition:      opacity 0.3s;
}

② イージング関数（timing-function）

重要ポイント：イージング関数はアニメーションの速度変化を制御します。 ease・linear・ease-in・ease-out・ ease-in-out の5つのキーワードと cubic-bezier()・steps() を理解しましょう。

🎮 イージング ビジュアライザ

「再生」ボタンで全イージングを同時に比較しよう
/* イージング関数の種類 */
.ease        { transition-timing-function: ease; }
/* 最初ゆっくり → 加速 → 最後ゆっくり（デフォルト）*/

.linear      { transition-timing-function: linear; }
/* 一定速度 */

.ease-in     { transition-timing-function: ease-in; }
/* ゆっくり始まり → 加速（入場に不向き）*/

.ease-out    { transition-timing-function: ease-out; }
/* 速く始まり → 減速（入場に最適）*/

.ease-in-out { transition-timing-function: ease-in-out; }
/* ゆっくり始まり → 加速 → ゆっくり終わる */

/* cubic-bezier：カスタムイージング */
.bounce {
  transition-timing-function: cubic-bezier(0.68, -0.55, 0.27, 1.55);
  /* バウンス効果 */
}
.spring {
  transition-timing-function: cubic-bezier(0.34, 1.56, 0.64, 1);
}

/* steps：ステップアニメーション */
.steps {
  transition-timing-function: steps(5, end);
  /* 5段階でジャンプ */
}

③ transform（変形）

重要ポイント：transform は要素を移動・拡大縮小・回転・傾斜させます。 GPU アクセラレーションが効くため、 top/left より transform: translate() の方が パフォーマンスが良いです。

/* translate：移動 */
.box { transform: translateX(50px); }
.box { transform: translateY(-20px); }
.box { transform: translate(50px, -20px); }

/* scale：拡大縮小 */
.box { transform: scale(1.5); }     /* 1.5倍 */
.box { transform: scaleX(2); }     /* 横方向2倍 */
.box { transform: scale(0.5, 2); } /* 横0.5倍・縦2倍 */

/* rotate：回転 */
.box { transform: rotate(45deg); }
.box { transform: rotate(-90deg); }

/* skew：傾斜 */
.box { transform: skewX(20deg); }
.box { transform: skew(20deg, 10deg); }

/* 複数の transform を組み合わせる */
.box {
  transform: translate(50px, -20px) scale(1.2) rotate(15deg);
  /* 左から順に適用される点に注意 */
}

/* transform-origin：変形の基準点 */
.box { transform-origin: top left; }   /* 左上を基準 */
.box { transform-origin: 50% 50%; }    /* 中心（デフォルト）*/
.box { transform-origin: 0 100%; }     /* 左下を基準 */

④ @keyframes と animation

重要ポイント：@keyframes でアニメーションの各フレームを定義し、 animation プロパティで要素に適用します。 transition と違い、トリガーなしで自動再生できます。

🎮 @keyframes アニメーション プレイグラウンド

1s
✦
← コントロールを操作してコードを確認しましょう
/* @keyframes の定義 */
@keyframes slide-in {
  from {
    transform: translateX(-100%);
    opacity:   0;
  }
  to {
    transform: translateX(0);
    opacity:   1;
  }
}

/* パーセントで細かく制御 */
@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50%      { transform: translateY(-30px); }
}

/* animation プロパティで適用 */
.element {
  animation: slide-in 0.5s ease-out forwards;
  /* name duration timing fill-mode */
}

/* animation の各プロパティ */
.element {
  animation-name:            bounce;
  animation-duration:        1s;
  animation-timing-function: ease-in-out;
  animation-delay:           0.2s;
  animation-iteration-count: infinite;
  animation-direction:       alternate;
  animation-fill-mode:       both;
  animation-play-state:      running;
}

/* animation-fill-mode の値 */
/* none      → デフォルト */
/* forwards  → 終了後に最終フレームを維持 */
/* backwards → 開始前に最初のフレームを適用 */
/* both      → forwards + backwards */

⑤ パフォーマンスと will-change

will-change

/* will-change：ブラウザにアニメーションを事前通知 */
.animated {
  will-change: transform;
  /* GPU レイヤーを事前に作成 → スムーズなアニメーション */
}

/* 複数プロパティ */
.animated {
  will-change: transform, opacity;
}

/* ⚠️ 注意点 */
/* 常に設定するのは NG（メモリを消費）*/
/* アニメーション直前に JS で追加するのが理想 */
/* アニメーション終了後は auto に戻す */

/* JavaScript での制御 */
el.addEventListener('mouseenter', () => {
  el.style.willChange = 'transform';
});
el.addEventListener('animationend', () => {
  el.style.willChange = 'auto';
});

prefers-reduced-motion

prefers-reduced-motion とは： ユーザーが OS でアニメーション削減を設定している場合に アニメーションを無効化するメディアクエリです。 アクセシビリティの観点から必須の対応です。 
/* prefers-reduced-motion：アニメーション削減設定への対応 */
@media (prefers-reduced-motion: reduce) {
  /* アニメーションを無効化 */
  * {
    animation-duration:        0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration:       0.01ms !important;
  }
}

/* 個別に対応する方法 */
.spinner {
  animation: spin 1s linear infinite;
}
@media (prefers-reduced-motion: reduce) {
  .spinner { animation: none; }
}

/* no-preference：アニメーションを許可している場合のみ適用 */
@media (prefers-reduced-motion: no-preference) {
  .hero {
    animation: fade-in 0.8s ease-out;
  }
}

/* JavaScript での確認 */
const prefersReduced =
  window.matchMedia('(prefers-reduced-motion: reduce)').matches;

アニメーション パフォーマンス Tips

推奨非推奨理由
transform: translateX() left / margin-left transform は GPU で処理されレイアウトを再計算しない
opacity visibility / display opacity は GPU 合成のみで高速
transform: scale() width / height width/height 変更はリフロー（再レイアウト）を引き起こす
特定プロパティを指定 transition: all all は不要なプロパティも監視してパフォーマンス低下
will-change: transform（適切に） 全要素に will-change 過剰な will-change はメモリを圧迫
💡 GPU アクセラレーションが効くプロパティ：
transform・opacity・filter・will-change

これらはコンポジット層で処理されるため、 レイアウト（リフロー）や描画（リペイント）を引き起こさず高速です。
